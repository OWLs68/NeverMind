### Блок 3 — Ключові UX-розширення

- **Inbox стрічка редизайн** (затверджено 30.03 + уточнення 01.04) — компактні картки з кольоровою крапкою + датові сепаратори ("СЬОГОДНІ", "ВЧОРА", "25 БЕР.") + закріплені нагадування зверху (🔴 критичні за 3 дні, 🟠 важливі за день). Тап → розгортання тексту на місці. Повний опис у `_archive/FEATURES_ROADMAP.md` секція "🗂️ Inbox стрічка — редизайн".
- **Динамічний розпорядок дня** (затверджено 10.04) — ключі у `nm_routine` за датами (`'2026-04-10': [...]`) замість шаблонних днів тижня (`mon: [...]`). Автододавання блоку у розпорядок коли юзер створює задачу/подію/нагадування з часом. Детектор конфліктів `_detectTimeConflict(date, time)` + `clarify_schedule_conflict` коли накладання. Модалка "Розпорядок" з вкладками по датах поточного тижня + навігація між тижнями. **Передумова: Блок 1 (`set_reminder` фікси).**
- **G4 Shadowing** — конфлікти розкладу: календар > розпорядок. "09:00 Робота" (розпорядок) + "09:00 Стоматолог" (календар) → AI проактивно пропонує перенос задач на після обіду.
- **G5 All-day events = "Фокус дня"** — події без часу показуються НЕ на таймлайні а зверху окремим блоком "Фокус дня: Приїзд мами".
- **G6 Дедуплікація нагадувань** — `linked_event_id` на ручному нагадуванні. Якщо збігається з існуючою подією → одне сповіщення замість двох (розпорядок + ручне).
- **Голосовий ввід** (затверджено 30.03) — Web Speech API (`SpeechRecognition`, `lang: 'uk-UA'`), кнопка мікрофона у полі вводу (іконка вже є в UI всіх барів). Тап → запис починається, текст у полі в реальному часі, пауза → зупинка автоматично. Безкоштовно, вбудовано у Safari iOS 14.5+ і Chrome.

### Блок 4 — Мозок OWL: інструменти AI (фундамент перед покращенням якості)

> Принцип "один мозок": якщо дані є в застосунку — AI має вміти їх створювати, читати, змінювати, видаляти. Без цих tools AI "імпровізує" — видаляє не те, плутає типи. Блок 5 (якість рішень) без цього не має сенсу.

- **4.11 `getAIContext` повне покриття** — розширити щоб AI бачив ВСІ типи даних: проекти (кроки, прогрес), події (календар), здоров'я (карточки, денні шкали), моменти дня. Без цього AI не може керувати даними навіть з правильними tools. Мала (1 сесія).
- **4.9 Повний набір CRUD tools** — додати відсутні інструменти в `INBOX_TOOLS`. Зараз пропущено: `delete_note`, `delete_finance`, `delete_memory_fact`, `edit_moment`/`delete_moment`, `edit_project`/`delete_project`/`add_project_step`/`complete_project_step`, tools для здоров'я (create/edit/delete health card, log daily scales). Середня (2 сесії).
- **4.10 Tool calling на всіх чат-барах** — зараз tool calling працює ТІЛЬКИ в Inbox. Решта вкладок (tasks, notes, finance, health, projects, evening) використовують старий `callAI()` без tools — суперечить "один мозок". Залежить від 4.9 (спочатку створити tools, потім підключити). Завершує Фазу 1.3. Середня (2-3 сесії).
- **G13 Brain Dump** — покращити парсинг суцільного тексту в Inbox. Чіп "Каша в голові" або просто абзац → AI розбирає на окремі записи (задачі, події, витрати, звички, нотатки) і показує результат. Частково працює зараз через масив actions — треба покращити промпт для суцільного тексту.

### Блок 5 — Мозок OWL: якість рішень

- **4.4 Relevance Scoring** — локальний бал 0-10 (без API) чи варто OWL говорити: повернувся після 30 хв = +3, дедлайн сьогодні = +3, чат відкритий = −3, ігнорував 3 табло поспіль = −5. Якщо бал < 5 → не генерувати. Прибирає "набридливу муху". Мала (1 сесія).
- **4.20 + 4.39 Negative Memory + Override** — консолідувати в одну фічу. Новий ключ `nm_negative_rules` зі структурою `{trigger, context, action: 'never_remind'}`. Авто-додавання правил коли юзер ігнорує тему 3+ разів. Кнопка UI "не нагадуй мені про це" на повідомленнях OWL. Передача правил у `getAIContext()`. Антиспам для OWL. Мала-середня (1 сесія).
- **4.6 Contextual Personality** — OWL адаптує не тільки тон а й поведінку: `planning` (ранок = план), `flow` (в потоці = мовчить), `struggling` (після невдачі = підтримка), `celebrating` (після перемоги = святкує), `reflecting`, `returning`. Функція `getContextMode()`. Мала (1 сесія).
- **G1 Тиша як модифікатор** — уточнення Judge Layer: тиша 60 хв НЕ самодостатній тригер. Підсилює інші сигнали, але без контенту (нема задач/звичок/дедлайнів) = 0 балів. OWL мовчить якщо нема про що — навіть після 3 годин тиші.
- **G2 Burst Detection** — антиспам серій. Юзер закрив 3+ задач за 2 хв → OWL мовчить, збирає дії у `burstQueue`. Через 3 хв тиші — одне резюме "🔥 5 задач за 20 хвилин" з чіпами. Один API-запит з масивом виконаного замість 3-5.
- **G3 Характери → різний поріг Judge Layer** — Тренер: поріг 4 (рідше говорить, різко). Партнер: поріг 2 (частіше на зв'язку). Ментор: поріг 3, активний на зміні фаз (ранок/вечір).
- **G10 Velocity tracking** — JS рахує темп юзера (скільки задач закриває до обіду, середнє за тиждень). Якщо сьогодні значно менше → Judge Layer отримує тригер `velocity-drop (+3)`. "Повільний день. Потрібна допомога?".
- **4.21 Verify Loop** — після дії коротко перевіряти результат а не просто "зроблено". Створив задачу → перевірив що вона в списку → підтвердив. Робить агента менш тупим.

### Блок 6 — Довершення

- **4.7 Мовчазний запис** — через Tool Calling AI сам вирішує коли `save_memory_fact()`. Частково через новий tool вже є (з 11.04) — треба допрацювати щоб спрацьовувало на ВСІХ завершених діях (закриття задачі → фіксує факт тихо). Залежить від 4.1 (Tool Calling) — вже зроблено.
- **4.8 Smart Notification Queue** — черга повідомлень OWL з пріоритетом замість перезапису табло. OWL хоче сказати 3 речі → показує по черзі. Критичні зверху. Середня.
- **4.41 Mirror Mode** — після задач/звичок з'являється мікро-чіп 😊/😐/😣. Через 3-4 дні OWL каже: "Останні 3 рази 'Підготовка до зустрічі' ти оцінював як 😣. Давай спростимо?" Jarvis-рівень розуміння без складних моделей.
- **4.43 Conversation Threading Фаза A** — локальний `nm_thread_index` за ключовими словами. JS витягує слова >4 букв (без stop-words) при створенні запису. Зберігає `{"київ": ["task_12", "event_45", "expense_88"]}`. При закритті задачі — перевіряє зв'язки: "Готель є! До поїздки 3 дні, квитки купив. Зібрати валізу?". Чистити >30 днів. Фаза B (повний holistic аналіз) — після Supabase.
- **Геймфікація** — обговорити деталі → імплементація. Концепція затверджена (`CONCEPTS_ACTIVE.md` → "Геймфікація"): маленькі бали щодня, великі бали за досягнення, видно у вкладці Я. Перед кодом обговорити з Романом: назви рівнів, точні числа балів, чи є соціальний елемент.

### Блок 7 — Адаптивність Android + desktop (в кінець, за рішенням Романа)

> Причина "в кінець": не блокує логіку застосунку, тільки UX на інших платформах. Але робити ДО Supabase бо потрібно для залучення юзерів на тест і не залежить від бекенду. Повний план і ризики — `_archive/FEATURES_ROADMAP.md` секція "📱💻 Адаптивний лейаут".

- **Фаза A — Android fit** (середня, 1-2 сесії) — тестування на реальному Android (Роман), ревізія `src/ui/keyboard.js` з `navigator.userAgent` розгалуженням iOS/Android, заміна hardcoded pixel'ів на `min()`/`max()`/`clamp()`, перевірка `color-scheme`, media queries `@media (min-width: 500px)`.
- **Фаза B — Desktop layout** (середня, 1-2 сесії) — `@media (min-width: 900px)` двоколонковий лейаут (лівий sidebar + правий content), клавіатурні шорткати (Ctrl+K фокус на Inbox, Ctrl+1-9 перехід на вкладку, Esc закрити модалку), hover-стани, курсор-friendly, макс. ширина ~700-800px щоб не розтягувалось на 4K.
- **Фаза C — PWA install для Android і desktop** (мала, 0.5 сесії) — перевірити `manifest.json` (іконки 192/512/maskable для Android), `display: standalone`, install prompt у Chrome Android і Chrome Desktop.

**Після цього — Supabase** (окрема велика фаза, див. секцію "🔒 After Supabase" нижче).

- **4.36 Fast Path / Deep Path** — 95% рішень: GPT-4o-mini + 4-5k токенів. 5% складних: GPT-4o або Claude коли confidence < 65%. Економить гроші.

## 🚫 Rejected — відхилені з причиною

- **BYOK (Bring Your Own Key)** — користувач сам реєструється у OpenAI і вставляє ключ. Суперечить UX "мінімальне тертя", зайвий крок в онбордингу. Замість — freemium + API на сервері Supabase.
- **Assistants API (OpenAI)** — суперечить принципу 4.22 Micro-orchestration (багато дешевих рішень на 4o-mini замість одного важкого виклику).
- **Поведінкова телеметрія** (трекінг часу на вкладках, ігнорування порад) — перебір. Ризик дратувати ("бачу, ти уникаєш фінансів"). Потребує дуже тонкого налаштування промптів.
- **Doubt Score** (лічильник сумнівів — створив і одразу видалив) — занадто нішево. Рідкісний кейс не варте окремої системи.
- **Лінгвістичні правила "робо-стиль"** ("Молоко. Нагадати?") — занадто сухо. OWL має характер (Partner/Coach/Mentor), не телеграф.
- **Видалити табло** — табло = ініціатива OWL, core фіча NeverMind.
- **Видалити чіпи** — мінімальне тертя = core принцип NeverMind.
- **Централізована БД-абстракція** (єдиний `db.js`) — пробували у старій флат-структурі до рефакторингу, зламало все одразу. Кожна вкладка працює з localStorage напряму.

**Anti-patterns (помилки яких уникати):**
- ❌ **"As an AI..."** — заборонити AI вибачатись і згадувати що він ШІ
- ❌ **Пасивна агресія** — "ти знову не виконав" ЗАБОРОНЕНО. Замість: "може дропнемо або розіб'ємо?"
- ❌ **Goldfish memory** — завжди передавати стан останніх 2-3 хвилин у промпт
- ❌ **API заради привітання** — мікро-реакції локально (масив фраз в JS), API тільки для смислових повідомлень
- ❌ **Набридливість** — OWL рідко говорить але завжди влучив

## 🔒 After Supabase — окремий блок

> Всі пункти що потребують серверу, pgvector, Edge Functions, Push або багатопристроєвої синхронізації.

**Порядок підключення Supabase (рекомендація GPT):**
1. Auth + RLS (авторизація + захист рядків)
2. Database
3. Realtime (синхронізація в реальному часі)
4. Edge Functions (серверні функції)
5. Push (сповіщення)
6. Storage (файли, аудіо)

**Відкладено на окреме обговорення ДО Supabase:**
- **SyncEngine архітектура** — як саме реалізувати offline-first sync (Варіант C за Gemini: localStorage синхронний, у фоні поштар відправляє на сервер). Окрема сесія проектування.
- **Деталізація freemium меж** — що саме у безкоштовній версії (скільки запитів AI на день? які вкладки доступні? базовий OWL без проактивності?), що у підписці. Впливає на UI онбордингу, Edge Functions rate limiting, систему обліку використання.
- **BroadcastChannel cleanup** — зняти `localStorage.setItem` override після перевірки що BroadcastChannel покриває всі кейси. Низький пріоритет — працює як є.

**Фічі що потребують серверу:**

- **4.9 Pattern Detection** — щотижневий аналіз: "Найпродуктивніший у вівторок 10-12", "Коли спиш <7 год — наступного дня −40% задач". Edge Function аналізує `nm_habit_log2`, `nm_tasks`, `nm_health_log`, `nm_finance`, `nm_moments`. Результат: `nm_patterns`.
- **4.10 Jarvis Insights** — щотижневі картки з аналізом тижня. Генерується в неділю. `nm_weekly_insight`. Спеціальний дизайн (не просто текст на табло).
- **4.11 Фонова рефлексія** — Edge Functions: коли юзер не в додатку, Jarvis аналізує дані, готує ранковий бриф, оновлює Day State, шукає патерни.
- **4.12 Push-сповіщення з контекстом** — Jarvis стукає навіть коли додаток закритий. VAPID-ключі, SW push handler.
- **4.13 Emotional Intelligence по поведінці** — аналіз не слів а ДІЙ: зайшов-погортав-вийшов = демотивація, 5 задач за годину = потік, короткі різкі повідомлення = роздратований. `nm_behavior_signals`.
- **4.14 Predictive Actions** — OWL пропонує ДО того як попросять: понеділок зранку → "Плануємо тиждень?".
- **4.15 Implicit Feedback Loop** — ігнор табло 3 рази → `save_memory("Не любить питання під час роботи")`. Ніколи не натискає чіпи з емодзі → перестати їх генерувати.
- **4.16 Трирівнева пам'ять (повна)** — Working Memory (10 реплік) → Day State (саммарі дня) → Core Memory (JSON по категоріях). Серверне оновлення через Edge Functions.
- **4.17 API-ключ на сервері** — замість `nm_gemini_key` у localStorage. Ключ на Supabase, запити через Edge Function.
- **4.18 Єдина пам'ять між пристроями** — один Jarvis на телефон/комп'ютер/планшет.
- **4.23 Intent Graph** — зв'язати задачі, звички, події, фінанси в одну карту НАМІРІВ. OWL бачить не записи а цілі. Потребує БД для зв'язків.
- **4.24 Planner / Executor / Reviewer** — розділити ролі AI. Потребує серверу.
- **4.26 Belief Graph** — кожен факт має `source`, `confidence`, `age`, `expiresAt`, `lastConfirmed`. OWL знає наскільки вірить у факт.
- **4.32 Voice-first (Realtime API)** — OpenAI Realtime API: WebRTC/WebSocket, voice-to-voice без STT/TTS ланцюжка.
- **4.37 IndexedDB замість localStorage** — коли дані виростуть за ліміт 5-10MB.
- **4.38 Offline-first sync** — local-first залишається священним. Supabase тільки для sync + auth + push. Last-write-wins + user version vector.
- **4.43 Conversation Threading Фаза B** — повний holistic аналіз з Pattern Detection (Фаза A локальна — у Блоці 6).
- **4.47 RAG через `search_facts(query)` tool** — замість запихати всі факти у `getAIContext()`, дати AI інструмент `search_facts(query)`. AI сам шукає потрібні факти. Після pgvector — embeddings + семантичний пошук. Заміна тривіальна: замінити `formatFactsForContext()` дамп на tool definition.
- **i18n — дві мови** — англійська + ще одна. `src/core/i18n.js` з `t('key')`, `nm_settings.lang`, перемикач у Settings UI. ~3-4 сесії.
