## ✅ Закриті (читай перед змінами — щоб не повторити)

### B-01 — `app-notes.js` — Undo повертає нотатку в кінець — ВИПРАВЛЕНО 26.03.2026

**Симптом:** після видалення нотатки та натискання Undo — нотатка з'являлася в кінці списку замість оригінальної позиції
**Причина:** undo зберігав числовий індекс (`noteOrigIdx`). Якщо між видаленням і undo масив нотаток змінювався (нова нотатка через AI/Inbox) — індекс зсувався. При `noteOrigIdx >= n.length` нотатка потрапляла в кінець масиву через `Math.min(noteOrigIdx, n.length) = n.length`
**Очікувана поведінка:** нотатка повертається на ту саму позицію відносно сусідніх нотаток (між тими ж нотатками що були до і після неї)
**Виправлення:** замість числового індексу зберігаємо `predecessorId` — ID нотатки яка стояла перед видаленою. При undo шукаємо попередника по ID і вставляємо після нього. Якщо попередника немає (нотатка була першою) — вставляємо на початок. Виправлено в `deleteNote` і `noteSwipeEnd`.

### B-02 — `app-inbox.js:597` — `==` замість `===` при порівнянні habit_id — ВИПРАВЛЕНО 26.03.2026

**Симптом:** потенційні хибні збіги при пошуку звички по ID
**Причина:** `habits.find(h => h.id == habitId)` — нестрогий оператор рівності. Якби AI повернув ID як рядок замість числа, або якби ID мав значення `0`/`false`/`null` — оператор `==` міг би дати хибний збіг
**Очікувана поведінка:** пошук звички по ID завжди точний, без type coercion
**Виправлення:** `h.id == habitId` → `h.id === habitId` в функції `processCompleteHabit`

### — SW кеш `nm-v1` — ВИПРАВЛЕНО 25.03.2026

**Симптом:** після деплою застосунок у браузері не оновлювався
**Причина:** Service Worker мав захардкоджену назву кешу `nm-v1` — ніколи не змінювалась, тому браузер не бачив нову версію
**Виправлення:** GitHub Actions тепер генерує унікальний `CACHE_NAME = nm-YYYYMMDD-HHMM` при кожному деплої

---

### — Мітка деплою захардкоджена — ВИПРАВЛЕНО 25.03.2026

**Симптом:** мітка версії в Inbox (наприклад `v66 · 25.03 15:42`) не оновлювалась після деплою
**Причина:** рядок був захардкоджений в коді, не автоматизований
**Виправлення:** GitHub Actions тепер автоматично замінює мітку в `id="deploy-version"` реальним часом деплою

---

### — Quit-звички: зрив обнуляв прогрес — ВИПРАВЛЕНО 25.03.2026

**Симптом:** один зрив (relapse) в quit-звичці обнуляв всі метрики — streak, freedomDays тощо
**Причина:** логіка не розрізняла "скинути streak" і "скинути все"
**Очікувана поведінка:** `freedomDays` — ніколи не скидається (загальна кількість днів без звички). `streak` — скидається при зриві. Зриви зберігаються в `relapses[]` для тренду
**Виправлення:** розділено логіку freedomDays (акумулятивні) і streak (скидається при зриві)
