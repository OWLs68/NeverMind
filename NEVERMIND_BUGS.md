### B-13 — Апостроф у `onclick` чіпів ламає клік

**Пріоритет:** 🔴 HIGH
**Файл:** `src/owl/board.js:137-140` (і потенційно `src/owl/inbox-board.js` — перевірити)
**Знайдено:** 2026-04-05, Gemini 3 Pro (аудит через `/gemini`), підтверджено Claude Opus 4.6

**Опис:** У `renderTabBoard()` чіпи рендеряться з інлайн `onclick` що містить текст чіпа:
```javascript
const s = escapeHtml(c).replace(/'/g, '&#39;');
return `<div class="owl-chip" onclick="owlChipToChat('${tab}','${s}')">${escapeHtml(c)}</div>`;
```

Спроба екранування апострофа через `&#39;` **не працює** — HTML-парсер браузера декодує entity назад у `'` **перед** тим як JS-парсер побачить код атрибута.

**Чому це баг:** Якщо текст чіпа містить апостроф, браузер генерує невалідний JS:
```
onclick="owlChipToChat('tasks','п'ять задач')"
                                  ↑
                         SyntaxError тут
```

**Конкретні кейси які гарантовано зламаються:**
- Українські слова з апострофом: **п'ять, м'ясо, сім'я, здоров'я, ім'я, об'єм**
- Англійські фрази: **I'm, don't, can't**

**Пропоноване рішення:** Перенести текст чіпа в `data-*` атрибути і повісити один делегований обробник:

```javascript
// Рендер у renderTabBoard():
const chipsHTML = (msg.chips || []).map(c =>
  `<div class="owl-chip" data-chip-tab="${escapeHtml(tab)}" data-chip-text="${escapeHtml(c)}">${escapeHtml(c)}</div>`
).join('');

// Один делегований обробник (після присвоєння innerHTML):
chipsEl.addEventListener('click', (e) => {
  const chip = e.target.closest('.owl-chip');
  if (!chip) return;
  const t = chip.dataset.chipTab;
  const txt = chip.dataset.chipText;
  if (chip.classList.contains('owl-chip-speak')) {
    openChatBar(t);
  } else {
    owlChipToChat(t, txt);
  }
});
```

`dataset` безпечно зберігає будь-який текст — HTML-парсер не виконує його як JS.

**Перевірити також:** `src/owl/inbox-board.js` — можливо той самий паттерн використовується для чіпів Інбоксу.

**Тест-кейс ручний (після фіксу):**
1. Створити чіп з текстом `п'ять задач` через консоль: `saveTabBoardMsg('tasks', {text:'тест', chips:["п'ять задач"], ts:Date.now()})`
2. Тапнути на чіп
3. ✅ Має: відкрити чат-бар вкладки tasks з текстом "п'ять задач"
4. ❌ До фіксу: `SyntaxError: missing ) after argument list` у консолі, клік не спрацьовує

### B-14 — `includes()` в chips.js дає false positives

**Пріоритет:** 🟡 MEDIUM (дизайн-smell, не критично поки AI дотримується правила "3 слова")
**Файл:** `src/owl/chips.js:26-33`
**Знайдено:** 2026-04-05, Gemini 3 Pro

**Опис:** `owlChipToChat()` розрізняє навігаційні чіпи від текстових через `String.prototype.includes()`:
```javascript
const navMap = [
  { patterns: ['задач', ...], tab: 'tasks' },
  ...
];
for (const nav of navMap) {
  if (nav.patterns.some(p => lower.includes(p))) {
    switchTab(nav.tab);
    return;
  }
}
```

**Чому це баг:** Будь-який чіп що **містить** підрядок `"задач"` перекине користувача на вкладку замість відправки в чат:
- `"обговорити задачі"` — користувач очікує чат, отримує навігацію
- `"нотатки по проекту"` — хибний кидок на notes замість projects

**Поки працює** бо промпт до AI обмежує чіпи до 3 слів. Але фундаментально ненадійно.

**Пропоноване рішення:** Явна типізація чіпів у відповіді AI:
```json
{
  "chips": [
    {"label": "Закрити задачі", "action": "nav", "target": "tasks"},
    {"label": "Пізніше", "action": "chat", "text": "нагадай завтра"}
  ]
}
```

**Що треба змінити:**
1. Промпт у `src/owl/proactive.js` (`generateTabBoardMessage`) — оновити формат JSON
2. Промпт у `src/owl/inbox-board.js` — той самий
3. `saveTabBoardMsg()` — зберігати новий формат
4. `renderTabBoard()` у `board.js` — передавати `action` в dataset
5. `owlChipToChat()` у `chips.js` — розгалуження по `action`, прибрати `navMap`
6. **Міграція:** fallback для старих `chips:[string]` у localStorage

### B-15 — `setTimeout(100)` у chips.js — тендітна синхронізація

**Пріоритет:** 🟢 LOW
**Файл:** `src/owl/chips.js:43-52`
**Знайдено:** 2026-04-05, Gemini 3 Pro (частково підтверджено)

**Опис:** Після `openChatBar(barTab)` відправка повідомлення робиться через жорсткий `setTimeout(..., 100)`:
```javascript
openChatBar(barTab);
// ... input.value = text
setTimeout(() => { sendToAI(); }, 100);
```

**Чому це (потенційно) баг:** 100 мс — магічне число без гарантій. На повільному iPhone з анімацією може не вистачити.

> **Примітка Claude:** Gemini помилково описав що `getElementById` повертає null — насправді він викликається синхронно до setTimeout, і в NeverMind input'и живуть у статичному `index.html`. Реальна потенційна проблема — race з пересіканням рендерів OWL.

**Рішення:** зробити `openChatBar()` асинхронним (Promise) і використати `await` замість `setTimeout`. **Робити тільки в рамках окремого рефакторингу чат-бара** — зачіпає всі caller'и.

## 📋 wontfix / Галюцинації AI

### "Синтаксична помилка `<msg.id>`" у `board.js:125` (05.04.2026)

Gemini під час аудиту повідомив про "критичну синтаксичну помилку" у конструкції:
```javascript
const ago = Date.now() - (msg.ts || msg.id || Date.now());
```
Стверджуючи що там є `<msg.id>` з кутовими дужками. **Це галюцинація** — у реальному коді дужок немає, це валідний JS-ланцюжок з `||` fallback.

**Урок:** завжди перевіряти точну цитату Gemini проти реального коду перед виправленням. Не довіряти "критичним синтаксичним помилкам" без верифікації.
