# Шпаргалка: як говорити з Claude

> Для Романа. Без технічних слів — тільки фрази що працюють.

---

## Головне правило

Скажи **"Роби"** — тільки тоді Claude чіпає код.
Без "Роби" — тільки обговорення.

---

## Коли щось зламалось

```
[скріншот або опис]
Ось що сталося. Як виправити?
```

Потім: **"Роби"**

---

## Коли хочеш нову функцію

```
Хочу щоб [що має відбуватись].
Наприклад: хочу щоб при натисканні на задачу відкривався чат з OWL.
```

Claude запропонує → ти кажеш **"Роби"** або **"Не те, хочу інакше"**

---

## Коли не розумієш що Claude каже

```
Поясни простіше. Без технічних слів.
```

або

```
Що це означає на практиці?
```

---

## Коли Claude зробив щось не те

```
Не те. Поверни як було.
```

або

```
Зробив не те що я мав на увазі. Я хотів [поясни].
```

---

## Коли хочеш перевірити стан проекту

```
Що зараз зламано? Що наступне?
```

Claude прочитає STATUS.md і відповість.

---

## Коли хочеш зрозуміти як щось працює

```
Поясни як працює [назва функції або вкладки].
Простими словами.
```

---

## Коли маєш ідею але не знаєш чи реально

```
Чи реально зробити [ідея]?
Які мінуси?
```

Claude має сперечатись якщо ідея слабка — це нормально.

---

## Коли хочеш зробити кілька речей

```
Зроби це по черзі:
1. [перше]
2. [друге]
3. [третє]
```

Claude зробить одне, скаже що готово, потім наступне.

---

## Коли Claude запитує "Роби?"

Це означає: план готовий, чи починати?

- **"Роби"** → починає
- **"Ні, хочу інакше"** → обговорюємо далі
- **"Стоп"** → зупиняється

---

## Чого не треба казати

| Замість цього | Кажи так |
|---------------|----------|
| "Можеш зробити XYZ?" | "Зроби XYZ. Роби." |
| "А що якщо..." | "Хочу щоб... Роби." |
| "Спасибі, все добре" | (просто наступне завдання) |
| Довге пояснення технічно | Опиши що бачиш або що хочеш |

---

## Золоте правило

**Описуй що бачиш або що хочеш — не як це зробити.**
Технічна частина — завдання Claude.

> Погано: "Зміни функцію renderTasks щоб вона сортувала за датою"
> Добре: "Хочу щоб задачі з дедлайном сьогодні були зверху"
