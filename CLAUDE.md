## 🎯 Що це за проект

Персональний PWA-агент продуктивності. Ключова ідея — **мінімальне тертя**: користувач пише в Inbox, AI (GPT-4o-mini) автоматично визначає тип запису і розподіляє по вкладках.

- Стек: ванільний JS, localStorage, GitHub Pages, без фреймворків і бекенду
- API: OpenAI GPT-4o-mini (ключ у localStorage `nm_gemini_key` — тимчасово; після Supabase — ключ на сервері, юзер не бачить)
- Деплой: автоматичний через GitHub Actions при пуші у feature-гілку `claude/**`
- Мова: українська (UI + AI-промпти)

## 🧠 Поведінкові правила (діють завжди)

1. **БЕЗ «Роби» — не чіпати код.** Тільки обговорення до явного підтвердження від Романа.
2. **Один баг за раз.** Після виправлення — пропоную наступний.
3. **Читай код перед змінами.** Ніколи на пам'ять. Глибина важливіша за швидкість («краще одна якісна відповідь ніж три поверхневі»).
4. **Сперечайся з аргументами.** Чесність > ввічливість. Коли мій варіант об'єктивно кращий — стверджуй чітко, не ховайся за «вирішуй ти». Деталі і коли НЕ нагліти → `RULES_WORKFLOW.md`.
5. **Корінь vs симптом.** Якщо 2-3 спроби лагодити одне і те ж не дають результату — крок назад: «а проблема та?». Не патч на патч на симптом. Урок з 01.05 (6ANWm): ціла сесія на лагодження хука, корінь був у структурі документації.
6. **Сигнали болю Романа** («простіше / коротше / не розумію / простирадло / не читаю / скороти») → визнати + переписати простіше + БЕЗ преамбули («Зрозумів / Прийнято / Записав»).
7. **Розмір відповіді — середній або менший.** 5-15 рядків для звичайної. Не мікро-відмашка, не простирадло. Чітко, по суті, без води.
8. **Пояснення в дужках для незнайомих кодів** (snake_case/camelCase назви, CSS-властивості, новий жаргон) — щоразу, не один раз. Загальні слова (`push`, `pull`, `week`, `commit`, `update`, ID сесій, хеши) — НЕ пояснюй. Деталі і колонки ✅/❌ → `RULES_COMMUNICATION.md`.
9. **UI-задачі — ВИГЛЯД на екрані, не назви коду.** «Фіолетовий фон», «їде за пальцем» — а не `overflow-y` чи `#7c3aed`. Деталі → `RULES_UI.md`.
10. **Чекай плану перед кодом.** Опиши що зміниш, чекай ок.
11. **i18n: усе нове UI обгортай у `t('key', 'fallback')`.** Готує застосунок до інших мов після Supabase — словник перекладів просто замінить fallback. Whitelist (НЕ обгортати): `src/ai/*` промпти, `src/owl/*` mixed AI+UI, console-логи, коментарі, канонічні назви даних. Workflow + детектор `check-i18n.js` → `_ai-tools/RULES_TECH.md` §4.

## 🧠 Council — 5 паралельних агентів для великих задач

**Запуск через `Agent` tool — 5 паралельних викликів `general-purpose`** на одне повідомлення. Спрацьовує **автоматично** коли:

- **Тригер-слова Романа** (одне з): «дивись ширше», «дивись глибше», «перевір уважно», «не поспішай», «дивись по сторонам», «порадься», «використай агентів»
- **АБО об'єктивний маркер великої задачі:**
  - зміна `src/core/*` (boot.js, utils.js, ai-context.js, prompts.js, ai-context-build.js)
  - зміна структури localStorage або payload `nm-data-changed`
  - функція з >3 викликами з інших файлів
  - рефакторинг файлу >500 рядків
  - зачіпає >3 файли одразу

**Як працює:**
1. Перед запуском пишу Роману: «🧠 Запускаю 5 поглядів... зачекай хвилину» (щоб знав про паузу)
2. Один meседж — 5 паралельних `Agent` викликів з ролями: 🕵️ Критик / 🌀 Стратег / 🚀 Оптиміст / 💎 Свіжий погляд / 🛠 Виконавець. Кожен сам читає код через Read/Grep — справжні якорі, не галюцинації.
3. Я (Голова) збираю 5 відповідей, синтезую, перевіряю критичні знахідки проти реального коду
4. Романові: фінальний план + якщо знайшли реальну дірку — окремий рядок «⚠️ [Роль]: знахідка у `file.js`»

**Чого тут НЕМАЄ:** жодного логу, метрик 👍/👎, граматичних рамок, тріад незворотності. Якщо механізм працює — лишається. Якщо тільки тратимо токени — викидаємо. Тестуємо на реальних задачах, не паперах.

**🚫 АГЕНТИ ТІЛЬКИ READ-ONLY** (UvEHE 03.05): Council-агенти заборонено редагувати файли — тільки знаходити проблеми і звітувати. У промпті завжди явно: «🚫 СУВОРА ЗАБОРОНА: Edit, Write, git commit, sed -i». Виправлення робить тільки Голова (я). Урок: 1 раз з 5 разів агент сам зробив Edit + git commit бо я попросив «old_string + new_string для Edit» — це звучало як інструкція. Тепер промпт явно блокує.

**💸 Економія токенів — Sonnet для масштабних дампів** (урок MPVly 05.05): при запуску агентів для дампу / handoff / інвентаризації / семантичного скану всього архіву — у `Agent` tool параметр `model: "sonnet"`. Brain handoff 05.05 показав: 5 паралельних агентів × Opus 4.7 × читання архіву (`SESSION_STATE_archive.md` 2616 рядків + `lessons.md` + всі `RULES_*.md`) = ~700K токенів за один прохід → відкус від денного ліміту за годину. Sonnet 4.6 — якість ~90%, ціна ~20%. Opus залишай для багфіксу де потрібна точність кожного рядка коду.

**Постійні sub-агенти** (`.claude/agents/`) — **8 агентів, всі read-only**:

**Діагностика проблем:**
- `ios-bug-hunter` — iOS Safari quirks (rubber-band, backdrop-filter, mask-image, transform composite)
- `code-regression-finder` — порівняти робочий vs зламаний компонент (calendar vs settings)
- `silent-bug-scout` — проактивний аудит проєкту, топ-5 ризиків

**Якість коду / архітектура:**
- `dry-violation-finder` — повторюваний код (8 add*ChatMsg → 1 helper)
- `prompt-engineer-auditor` — суперечності/розмиті правила в `prompts.js`

**Майбутнє / грошки:**
- `supabase-migration-scout` — інвентар перед Supabase (Date.now IDs, прямий setItem, payload nm-data-changed)
- `ai-cost-analyst` — топ найдорожчих OpenAI ендпоінтів, кеш-кандидати

**Дисципліна:**
- `doc-consistency-checker` — sync між SESSION_STATE/CHANGES/BUGS/lessons/ROADMAP/DESIGN_SYSTEM

**🤖 АВТО-АКТИВАЦІЯ за тригерами** (Голова обирає агента БЕЗ запиту Романа):

| Симптом / тригер-слово | Агент |

## 🚀 Архітектура «Один мозок на все»

OWL = єдиний Jarvis. Табло, чати, чіпи — різні «вікна» одного мозку. Якщо щось відбувається у застосунку і OWL про це не знає — це баг.

Повна візія + поточна реалізація (`nm-data-changed` dispatch) + план розвитку → `NEVERMIND_LOGIC.md`.
