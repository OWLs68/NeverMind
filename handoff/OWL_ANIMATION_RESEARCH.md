## 1. Що ми вже зробили (2 сесії: w3ISi + uDZmz)

**Зроблено у коді:**
- Базова інтеграція 5 PNG сови у `#owl-mascot-main` (стан `data-state`)
- Priority state-machine (`setOwlMascotState` у `src/owl/board.js`) — error>alert>thinking>greeting>idle. Ticket-лічильник + failsafe 30с.
- Кросфейд 400 мс між станами, float 4с (легке погойдування)
- `visibilitychange` → пауза анімацій коли PWA у фоні (економія батареї)
- `prefers-reduced-motion` виняток (доступність)
- Sprite-sheet для махання (6 кадрів) — **відкинуто**, кадри від Gemini не консистентні
- SVG-крило overlay поверх PNG-тіла для greeting — **відкинуто**, живе SVG на статичній PNG-сові ріже око

**PNG-ассети:**
- `owl-idle.png` і `owl-greeting.png` — нові від Романа (amber/brown, прозорий фон) ✅
- `owl-alert.png`, `owl-thinking.png`, `owl-error.png` — ще старі "здивовані" 🟡

**Висновок сесії uDZmz:** зробити flipbook — швидке перемикання `opacity`
між idle.png ↔ greeting.png 3×150мс при `data-state="greeting"`.

## 2. Де ми реально стоїмо (чесна оцінка)

Поточний підхід **"5 статичних PNG + CSS кросфейд + одне CSS псевдомахання"**
має фундаментальну стелю якості:

| Проблема | Чому не вирішиться без зміни підходу |
|---|---|
| 2 кадри greeting = не махання, а "моргання" | Потрібно 8-24 проміжних кадри між позами |
| Решта станів "мертві" (один статичний PNG) | Нема вторинної анімації — кліпання очей, дихання, нахил голови |
| 5 окремих PNG ~1 МБ кожна = ~5 МБ трафіку | Вектор або .riv це одразу розв'яже |
| Неможливо "перетекти" з idle у thinking плавно | Кросфейд бачимо як "заміна картинки", не як рух |

**9 комітів ітерацій** показали: кожен крок дає +5% якості, стеля десь **~6/10**.
Для Jarvis-рівня треба **9/10**. Поточний шлях туди не доведе.

## 3. Що показало дослідження (індустрія 2026)

**Ключове відкриття:** Duolingo **перейшли з Lottie на Rive** для сови Duo —
саме через те що ми зараз впираємось (статеві стани, інтерактивність, розмір).

**Порівняння трьох індустріальних підходів:**

| Параметр | CSS/PNG (де ми) | Lottie (JSON з After Effects) | **Rive** (.riv файл) |
|---|---|---|---|
| Розмір ассета | ~5 МБ (5 PNG) | ~240 КБ | **~16 КБ** |
| Розмір runtime (бібліотека) | 0 | ~250 КБ `lottie-web` | ~100 КБ `@rive-app/canvas` |
| FPS (частота кадрів) | ~60 (float) | ~17 на складних | **~60** (GPU) |
| Стани (state machine) | ❌ руками в JS | ❌ тільки сегменти | ✅ вбудовано — 1:1 до нашого priority |
| iOS Safari | ✅ | ✅ (Core Animation) | ✅ (Metal GPU) |
| Інтерактивність | Обмежена | ❌ тільки play/pause | ✅ бул/число input-и |
| Хто робить ассет | Художник + розробник | Дизайнер в After Effects | Дизайнер в Rive-редакторі |
| Вартість замовити | Безкоштовно (є) | $30-80 на Fiverr | **$50-200** на Fiverr |
| Час інтеграції | 0 | 2 години | 3-4 години |
| Де Duolingo зараз | — | (був раніше) | **✅ зараз** |

**Чому Rive переміг Lottie у цьому конкретному випадку:**
Lottie — це "згортка відео", ти можеш тільки прокрутити до кадру. Rive — це
"стани з переходами", ти кажеш "alert!" і він сам програє alert-анімацію +
плавний перехід назад в idle. Це **буквально те саме** що наш
`setOwlMascotState('alert', 12000)` — тому інтеграція 1:1.

## 4. Три шляхи вперед

### Tier 1 — Доробити поточний PNG-підхід (швидкий виграш, 1 сесія, $0)

**Що робимо:**
- Прибрати debug SVG-крило з `style.css` (дашована червона рамка)
- Прибрати `<svg class="owl-wing-overlay">` з `index.html`
- Замість нього — flipbook greeting: `@keyframes` що 3×150мс перемикає opacity
  між idle.png і greeting.png (використовуючи тільки свіжі PNG)
- **Додати шари вторинної анімації на ВСІ стани:**
  - Постійне "дихання" (`scale: 1 → 1.02 → 1` 3.5с cycle)
  - Кліпання очей раз на 4-6с (окремий прозорий overlay-div зверху)
  - Легкий `rotate(-1deg → 1deg)` при thinking (голова "гойдається в роздумах")
- Генерувати 3 нові PNG для alert/thinking/error у тому ж amber-стилі через ті
  ж промпти що дали гарні idle+greeting

**Підсумок:** ~7/10 якості. Максимум що PNG-підхід витисне.

### Tier 2 — Замовити Rive у дизайнера (як Duolingo, 1-2 тижні, $100-200)

**Що робимо:**
1. Написати ТЗ дизайнеру на Fiverr/Upwork: "5 станів сови з плавними переходами,
   amber палітра `#c2790a`, state machine Rive, експорт `.riv`". Можу написати
   повне ТЗ на 1 сторінку.
2. Передати йому нашу поточну сову як референс (стиль збережеться)
3. Отримуємо один файл `owl.riv` (~30-50 КБ)
4. Підключення у код:
   - `npm install @rive-app/canvas`
   - Замінити `<div class="owl-mascot">` з 5 `<img>` на `<canvas id="owl-rive">`
   - У `setOwlMascotState` замість `data-state=` — `stateMachineInput.value = 'alert'`
   - Priority-машина залишається як є, лише кінцевий виклик змінюється
5. `sw.js` кешує `.riv` як static asset (vs 5 PNG зараз)

**Переваги:**
- Файл **у 30 разів менший** (5 МБ → ~50 КБ)
- 60 FPS плавні переходи між будь-якими станами
- Майбутнє: додати емоції (радість коли закрив задачу, грусть коли пропустив звичку)
  = дизайнер додає стейт у Rive, розробник додає 1 рядок
- Duolingo-рівень якості

**Мінуси:**
- Потрібен бюджет (реальний варіант — $80-150 на Fiverr за 3-5 днів)
- 100 КБ WASM runtime (один раз завантажується, PWA кешує)
- Залежність від формату `.riv` (vendor lock але формат опенсорсний)

### Tier 3 — Безкоштовний Lottie з LottieFiles (запасний план, 2-3 години, $0)

**Що робимо:**
- Знайти готову безкоштовну сову на lottiefiles.com/free-animations/owl
- Підключити `@lottiefiles/dotlottie-web` (легший за `lottie-web`)
- 5 станів мапити на 5 часових сегментів у одній анімації (frame markers)

**Проблема:** я не знайшов жодної безкоштовної LottieFiles-сови яка має
відразу 5 потрібних нам станів у amber-палітрі. Більшість це "логотип махає
крилом 2 секунди". Результат буде 6/10 — не краще за поточний PNG, тільки інакше.

**Вердикт:** Tier 3 не рекомендую — витрата часу без реального покращення.

## 5. Моя рекомендація

**Стратегія: Tier 1 + Tier 2 паралельно.**

- **Сьогодні-завтра** (2-3 години кодерської роботи): реалізую Tier 1 без тебе —
  прибрати debug, flipbook + вторинні CSS-анімації. Вкладка виглядає живо
  **вже завтра** поки чекаємо дизайнера.
- **Паралельно** (твоя дія): я пишу ТЗ для дизайнера, ти розміщуєш на Fiverr
  або Upwork. За $100 через тиждень маємо Duolingo-рівень Rive.
- **Коли .riv прийде** (~1 сесія): я замінюю PNG-флоу на Rive. Priority-машину
  не чіпаємо, тільки кінцевий виклик.

**Чому не тільки Tier 1:** стеля 7/10 — це для Jarvis недостатньо. Ти хочеш
"справжнього компаньйона" — без Rive або аналога це не досяжно.

**Чому не одразу Tier 2 без Tier 1:** 1-2 тижні чекання дизайнера = весь цей
час сова буде у поточному "debug-стані з червоною рамкою". Tier 1 закриває цей
gap і дає мені рефакторинг структури щоб Rive-інтеграція була тривіальною.
