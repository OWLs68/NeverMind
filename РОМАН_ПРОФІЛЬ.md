## Як він працює

- **Тільки телефон** — MacBook 2014 занадто старий для Claude Code, тому телефон = єдиний девайс
- **Claude Code = деплой** — спеціально налаштував щоб не відкривати ноутбук. Кожен пуш = автодеплой
- Будь-яка поломка деплою або непорозуміння = **втрата годин**, не хвилин

## Як він комунікує

- **Короткі повідомлення** — очікує те саме у відповідь
- **"Роби"** — сигнал що обговорення завершено, можна писати код
- **Без "Роби"** — тільки обговорення, код не чіпати
- **Скріншоти** — основний спосіб показати баг або ідею
- **Питає "чому"** — хоче розуміти логіку, не просто результат

## Чого НЕ робити (стиль спілкування)

- **Не показувати блоки коду без потреби** — Роман не читає код, він бачить результат у застосунку
- **Не давати довгі технічні пояснення як першу відповідь** — спочатку коротко суть, деталі тільки якщо запитав
- **Не перевантажувати** — одна думка за раз. Баланс: достатньо контексту для рішення, не більше
- **Не змінювати не той елемент** — завжди перевіряти CSS-клас (клас — ідентифікатор стилю у CSS) через скріншот перед зміною розміру/позиції
- **Не погоджуватись якщо є сумніви** — чесність важливіша за ввічливість
- **Не використовувати фіолетовий колір** — Роман його не любить. Альтернативи: бурштиновий `#c2790a`, темно-сірий `#1e1040`, білий. Детальніше → `docs/DESIGN_SYSTEM.md`
