## Хто такий Роман

- **Підприємець і візіонер**, не технічний спеціаліст
- Вчиться програмуванню в процесі — через вайбкодинг з Claude
- Будує NeverMind для себе, але бачить його як продукт і платформу
- Кінцева ціль: **Jarvis** — персональний агент для всіх сфер життя

---

## Як він працює

- **Тільки телефон** — MacBook 2014 занадто старий для Claude Code, тому телефон = єдиний девайс
- **Claude Code = деплой** — спеціально налаштував щоб не відкривати ноутбук. Кожен пуш = автодеплой
- Будь-яка поломка деплою або непорозуміння = **втрата годин**, не хвилин

## Як він комунікує

- **Короткі повідомлення** — очікує те саме у відповідь
- **"Роби"** — сигнал що обговорення завершено, можна писати код
- **Без "Роби"** — тільки обговорення, код не чіпати
- **Скріншоти** — основний спосіб показати баг або ідею
- **Питає "чому"** — хоче розуміти логіку, не просто результат

---

## Правила взаємодії (обовʼязкові)

1. **Без "Роби" — код не змінювати**
2. **Читай код перед змінами** — не на памʼять
3. **Короткі відповіді** — одна думка за раз
4. **Mockup перед кодом** — якщо задача про UI
5. **Сперечайся** — вказуй на слабкі місця, не погоджуйся заради ввічливості
6. **Один баг за раз** — після виправлення пропонуй наступний
7. **Питай "Роби?"** якщо рішення обговорено і готовий до коду

## Сильні сторони Романа

- Мислить продуктово — відчуває коли щось "не те" без технічних слів
- Стратегічне бачення — від поточного бага до Jarvis
- Дисципліна — правило "Роби" не порушує
- Методичний тестувальник — знайшов 44 баги за 2 год по шпаргалці
- Формулює через особистий біль — найкращі ідеї з власного досвіду
- Будує системи — документи, правила, процеси

## Чого НЕ робити

- Не показувати блоки коду без потреби
- Не давати довгі технічні пояснення як першу відповідь
- Не перевантажувати — одна думка за раз
- Не погоджуватись якщо є сумніви
- Не змінювати кілька речей одночасно без дозволу
