## Що це за проект

**CSTL NEWS** — локальна медіа-платформа, під-проект екосистеми **Olyka Castle**.

**Olyka Castle** — головний бренд/організація Влада. Названо на честь Олицького замку (англ. "Olyka Castle") — головної пам'ятки містечка Олика, Волинська область, Україна. Всі під-проекти мають у назві "CSTL" (скорочення від Castle): Smoke_Castle, Kino_Castle, CSTL NEWS.

**CSTL NEWS** — єдине місце де житель Олики знаходить:
- Ексклюзивні місцеві новини (яких немає ніде в інтернеті)
- Відібрані новини Волині та України (не весь шум, тільки важливе)
- Афішу подій регіону (включно з Kino_Castle та іншими під-проектами)
- Розклад автобусів через Олику (актуальний)

**Стек (технічний набір):** ванільний JS (чистий JavaScript без фреймворків), localStorage (локальне сховище браузера), GitHub Pages (безкоштовний хостинг), без бекенду (серверної частини)

**Деплой (публікація):** автоматичний через GitHub Actions при пуші в будь-яку гілку

**Мова UI:** українська

## ВСІ ПРАВИЛА

### Процес роботи
- **БЕЗ "Роби" від Влада — не змінювати код.** Влад каже "Роби" — і тільки тоді.
- **ОДИН баг за раз.** Після виправлення — запропонуй наступний.
- **ЧИТАЙ код перед змінами.** Ніколи "на пам'ять".
- **НІКОЛИ не створювати правила самостійно.** Завжди спочатку запитати Влада.
- **Сперечайся** — вказуй на слабкі місця ідеї, не погоджуйся заради ввічливості.
- **Пропонуй кращі рішення** — Claude краще знає технічні можливості. Не просто виконуй, а активно пропонуй.
- **Mockup перед кодом** — якщо задача про UI, показати макет і отримати підтвердження.
- **Пояснюй свої дії** — після кожної дії коротко: що зробив, для чого, як це працює.

### Якість виконання
- **Якщо перша спроба не спрацювала — СТОП.** Перечитати код, зрозуміти ЧОМУ не спрацювало, тільки тоді наступна спроба.
- **Не губи попередні фікси** — при кожній зміні перевіряти що попередні виправлення збережені.
- **Перевіряй синтаксис** — після JS змін: `node --check файл.js`.

### Дані (контент сайту)
- Статті зберігаються в `data/articles.json` — редагується вручну або через RSS агрегатор
- Події зберігаються в `data/events.json`
- Розклад автобусів в `data/schedule.json`
- **НЕ використовувати localStorage для статей** — він тільки для конкретного браузера, не підходить для публічного контенту

### Документація (після кожної зміни)
- **Після КОЖНОЇ зміни** — оновити `_ai-tools/SESSION_STATE.md`. Не чекати кінця сесії.
- **Після структурних змін** — оновити `CLAUDE.md` (файлова структура, правила).
- **Після змін архітектури** — оновити `docs/ARCHITECTURE.md`.

### Деплой (публікація на сервер)
- **При кожному деплої** — змінити `CACHE_NAME` (назву кешу) у `sw.js` (формат: `cstl-YYYYMMDD-HHMM`). Використовуй `date` в терміналі.
- **bundle.js не комітити вручну** — він генерується автоматично GitHub Actions

### Що не можна змінювати без обговорення
- Структуру `data/*.json` файлів без міграції (оновлення) існуючих даних
- Порядок імпортів в `src/app.js`
- Логіку `setupSW()` в `src/core/boot.js`

### Чеклист аудиту (коли Влад каже "перевір роботу")
1. Перечитати всі змінені ділянки
2. `node --check файл.js`
3. Перевірити баланс `<div>` тегів в HTML
4. Шукати дублікати функцій
5. Шукати залишки старого коду і мертві змінні
6. Перевірити що всі нові обробники зареєстровані

## Файлова структура

| Файл | Відповідальність |
|------|-----------------|
| `index.html` | Весь UI. Один `<script src="bundle.js">` |
| `style.css` | Всі стилі |
| `sw.js` | Service Worker. **CACHE_NAME міняти при кожному деплої** |
| `bundle.js` | Згенерований esbuild. **Не комітити** |
| `build.js` | Конфіг esbuild (8 рядків) |
| `package.json` | Одна залежність: esbuild |
| `data/articles.json` | Масив статей |
| `data/events.json` | Масив подій |
| `data/schedule.json` | Розклад автобусів |
| `src/app.js` | Точка входу — імпортує всі модулі |
| `src/core/boot.js` | PWA setup, Service Worker |
| `src/core/utils.js` | formatTime, escapeHtml, showToast |
| `src/tabs/news.js` | Стрічка новин, фільтри |
| `src/tabs/events.js` | Афіша подій |
| `src/tabs/buses.js` | Розклад автобусів |
| `src/tabs/submit.js` | Форма подачі новини |

**Збірка:** `node build.js` → `src/app.js` → `bundle.js` (esbuild, IIFE формат)

## Система деплою

**Флоу:** пушиш будь-яку гілку → `deploy.yml` збирає bundle.js → деплоїть на GitHub Pages

**`deploy.yml` робить:**
1. `npm install` — встановлює залежності
2. `node build.js` — збирає bundle.js з src/
3. Комітить bundle.js і пушить у main
4. Деплоїть на GitHub Pages

**URL сайту:** `https://vshevchukkk.github.io/CSTL_NEWS/`
