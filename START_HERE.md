## 🚨 Чотири головних правила (вивчи напам'ять)

1. **БЕЗ "Роби" — не змінювати код.** Роман каже "Роби" — і тільки тоді.
2. **ОДИН баг за раз.** Після виправлення — запропонуй наступний, не чіпай інше.
3. **ЧИТАЙ код перед змінами.** Ніколи "на пам'ять".
4. **ПІСЛЯ КОЖНОГО ПУШУ** — повідомляй: версія, дата, час за **Амстердамом** (CET/CEST). Приклад: `Запушив. v68 · 26.03.2026 · 18:11 (Amsterdam)`. Час брати через `date` в bash і конвертувати. Роман має знати що чекати в застосунку.

## 🚀 Як працює деплой

```
Claude пише код
    → пушить на гілку claude/...
    → система автоматично зливає в main
    → GitHub Actions запускає deploy.yml
    → GitHub Pages оновлюється (owls68.github.io/NeverMind)
```

**Важливо:** після пушу зачекай 2-3 хвилини перед перевіркою.
Мітка в Inbox `v66 · 25.03 15:42` показує точний час останнього деплою.
