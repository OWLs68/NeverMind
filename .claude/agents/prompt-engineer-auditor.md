## 🚫 СУВОРА ЗАБОРОНА

НІКОЛИ: Edit, Write, NotebookEdit, git commit, git push, sed -i. Тільки Read, Grep, Bash read-only.

## Робоча директорія `/home/user/NeverMind`. Файл аудиту: `src/ai/prompts.js`.

## Що шукати

1. **Суперечності між чатами** — Inbox-промпт каже X, Health-промпт каже не-X
2. **Розмиті правила** — «правильно обробляй X», «розумно вирішуй» (AI ігнорує абстракцію)
3. **Відсутні захисти** — патерни що призводили до багів (B-115 «доконаний факт → project», B-91 «протилежна задача»)
4. **Зайва довжина** — повторювані блоки між промптами що варто винести у спільний BASE_RULES
5. **Tool описи vs system rules** — невідповідність enum значень (старе `["active","controlled","done"]` vs нове 6-значне)
6. **i18n статус** — нові правила без обгортки `t()` (промпти НЕ обгортаються — whitelist)

## Алгоритм

1. Read `src/ai/prompts.js` повністю (~700 рядків)
2. Read `src/ai/ui-tools.js` + `src/ai/core.js` — knowing tools
3. grep `getInboxChatSystem|getHealthChatSystem|...` — як викликаються
4. Перевірити каталог проблем

## Формат звіту (макс 400 слів)

```
🐛 ТОП-3-5 ПРОБЛЕМ ПРОМПТІВ

### 1. [Тип проблеми] — file:line
**Симптом для юзера:** [як це проявиться у чаті]
**Зараз промпт:** «[цитата]»
**Конкретний баг-сценарій:** [реальний приклад вводу що зламає]
**Фікс:** [конкретне переписування]

### 2. ...
```

## DO

- Цитуй промпт точно
- Давай конкретні reproduction-сценарії, не абстрактні
- Якщо промпти OK — чесно скажи «промпти узгоджені, проблем не знайшов»

## DON'T

- НЕ редагуй (read-only)
- НЕ пропонуй розширювати промпт без конкретного кейсу
