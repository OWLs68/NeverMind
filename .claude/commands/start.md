# /start — Швидкий старт нової сесії

**Коли використовувати:** перша команда у кожному новому чаті. Один виклик — і Claude готовий до роботи з повним контекстом проекту.

**Без аргументів.** Просто `/start`.

---

## Крок 1 — Прочитати обов'язкові файли (у цьому порядку)

1. **`START_HERE.md`** (у корені репо) — карта проекту, файлова структура, список скілів, перший орієнтир
2. **`_ai-tools/SESSION_STATE.md`** — поточний стан: версія, гілка, що робили в останніх сесіях, **відкрите на майбутнє**. Особлива увага до секції "⚠️ ДЛЯ НОВОГО ЧАТУ — найважливіше" на початку файлу
3. **`NEVERMIND_BUGS.md`** (у корені репо, НЕ в `_ai-tools/`) — відкриті баги з пріоритетами (🔴 критичні, 🟡 середні, 🟢 дрібні)
4. **`FEATURES_ROADMAP.md`** — **СТРАТЕГІЧНИЙ ПЛАН**. Дорожня карта фаз розвитку (зокрема секція "🧠 Мозок OWL" з фазами 4.2, 4.3, 4.20 ДО Supabase і всім що ПІСЛЯ Supabase). **Без цього файлу Claude не розуміє куди йде проект і починає вигадувати "власні" рішення які не відповідають концепції.** Читати щоб бачити: (а) що вже зроблено, (б) що в планах ДО Supabase, (в) що ПІСЛЯ Supabase, (г) загальну візію Jarvis як Reasoning Engine
5. **`CONCEPTS_ACTIVE.md`** — реалізовані концепції вкладок (Finance, Evening, Me, Projects, Gamification). Дає зрозуміти ПОТОЧНИЙ функціонал. Читати швидко — достатньо зрозуміти які концепції активні
6. **`РОМАН_ПРОФІЛЬ.md`** — **тільки якщо давно (більше тижня) не було сесій**, або якщо це перша сесія в цьому чаті. Хто такий Роман, як з ним комунікувати

**Винятки — НЕ читати:** `CLAUDE.md` (автоматично завантажується як project instructions, вже в контексті), `_archive/*` (історичні документи), `bundle.js` (згенерований), `_ai-tools/REFACTORING_PLAN.md` (історична довідка, рефакторинг завершено), `NEVERMIND_LOGIC.md` (базова концепція — timeless, читати тільки за потреби).

## Крок 2 — Перевірити гілку

Виконати:
```bash
git branch --show-current
git log --oneline -5
```

Зрозуміти: на якій гілці працюємо, що в останніх комітах (commit — одна збережена зміна в git).

## Крок 3 — Сформувати коротку відповідь Роману

Написати **4-6 рядків максимум** українською:

1. **Одне речення статусу:** "Прочитав. Проект: NeverMind v[версія], гілка [назва], остання робота — [що робили]."
2. **Найкритичніший баг** (якщо є в `NEVERMIND_BUGS.md` з 🔴 пріоритетом) — одне речення
3. **3-5 варіантів що робити далі** — коротко, нумерованим списком. Брати з секції "Відкрите на майбутнє" у `SESSION_STATE.md` або з відкритих багів. Наприклад:
   - `/fix B-XX` — виправити конкретний баг
   - Верифікація старих багів проти реального коду
   - Нова функція яку обговорювали
   - Доопрацювання інструментів (tools — інструменти/скіли)

4. **Запитання:** "Що робимо сьогодні?"

## Крок 4 — Чекати відповідь Романа

**НЕ** починати жодних змін у коді до явного "Роби" від Романа. Це правило №1 проекту (див. `CLAUDE.md` → "Процес роботи").
