# /mockup — ASCII макет UI перед кодом

Аргумент: назва фічі або екрану. Наприклад: `/mockup calendar-tab` або `/mockup task-card-redesign`

## Що зробити

1. Намалюй ASCII макет (текстове схематичне зображення) нового UI елемента або екрану
2. Покажи різні стани якщо є: порожній стан, заповнений, активний, помилка
3. Вкажи розміри якщо критично: ширина, висота, padding (відступи всередині), margin (відступи зовні)
4. Поясни логіку взаємодії: що відбувається при тапі, свайпі, скролі

## Формат макету

```
┌─────────────────────────┐
│  Назва екрану           │
├─────────────────────────┤
│  [елемент]   [кнопка]  │
│                         │
│  Текст контенту         │
│  підзаголовок           │
└─────────────────────────┘
```

## Після макету

Написати Роману: "Ось макет. Підтверджуєш — і тільки тоді починаю код."

Не починати писати код поки Роман не сказав "Роби".
