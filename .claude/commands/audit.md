# /audit — Повна перевірка роботи

> **Коли запускати:**
> - Після значимого фіксу/фічі (3+ змінених файли або змінений src/core/*)
> - Перед /finish (фінальний контроль)
> - При підозрі на регресію («щось дивно поводиться»)
>
> **Як читати звіт:** дивись секцію «🔴 Критичне». Якщо порожня — push безпечний.

---

## Pass 0 — Швидкі статичні чеки (~30 сек, без агентів)

1. **Перечитай всі змінені файли** — не по пам'яті, а реально читай код через Read
2. **`node --check файл.js`** — для кожного зміненого JS-файлу (один Bash)
3. **Баланс `<div>` тегів** — порахуй відкриваючі і закриваючі у змінених частинах index.html
4. **Дублікати функцій** — `grep "^function NAME\|^export function NAME"` для імен з diff
5. **Мертвий код** — закоментовані блоки, невикористані змінні
6. **Розмір файлів** — `wc -l` змінених. >1200 рядків — попередь, >1500 — стоп, обговорити розбиття

Якщо щось знайдено — фіксу одразу перш ніж запускати Pass 1+.

---

## Pass 1 — Council аудит (паралельно, ~2-3 хв)

Запусти агентів **паралельно одним повідомленням** (вибери ті що матчать характер змін):

**Якщо змінювались `src/owl/`, `src/tabs/`, `src/core/`:**
- `silent-bug-scout` — топ-5 прихованих ризиків у проекті
- `code-regression-finder` — порівняння робочого vs зламаного компонента (якщо «X працює, Y ні»)

**Якщо змінювались модалки / CSS / UI на iOS:**
- `ios-bug-hunter` — Safari quirks (bfcache, backdrop-filter, transform, mask-image)

**Якщо змінювались `src/ai/prompts.js` АБО додано/змінено tool:**
- `prompt-engineer-auditor` — суперечності правил, AI-галюцинації, нечіткі правила

**Якщо змінювались localStorage ключі / `nm-data-changed` payload / Date.now() ID:**
- `supabase-migration-scout` — інвентар проблем перед Supabase міграцією

**Якщо чіпали структуру 8 add*ChatMsg / повторюваний код:**
- `dry-violation-finder` — копіпаст 5+ рядків у 3+ файлах

**Завжди в кінці:**
- `doc-consistency-checker` — sync між SESSION_STATE/CHANGES/BUGS/lessons/ROADMAP/DESIGN_SYSTEM

**Правила запуску:**
- Мінімум 2, максимум 5 агентів за виклик (rate limit)
- Усі агенти **read-only** — у промпті явно «🚫 СУВОРА ЗАБОРОНА: Edit, Write, git commit, sed -i»
- Кожен агент сам читає файли через Read+Grep — не передавати готові висновки

## Pass 2 — Синтез + рішення

Збираю результати Pass 0 + Pass 1, формую звіт:

```
🔴 КРИТИЧНЕ (треба фіксити перед push):
  [N] [заголовок] — [файл:рядок] — [агент-знайшов]

🟡 СЕРЕДНЄ (можна push, фіксити окремою фазою):
  ...

🟢 МІНОРНЕ (для довідки, не блокер):
  ...

✅ ЧИСТО (агенти підтвердили):
  - silent-bug-scout: PASS
  - prompt-engineer-auditor: PASS (5 з 5 правил консистентні)
  - ...
```

Після кожного знайденого 🔴 — пропонуй конкретний фікс (file:line + before/after Edit).

---

## Pass 3 — Post-fix re-audit (якщо були 🔴 фікси у Pass 2)

Patern сесії RGisY 04.05: **Phase 11 → 6 регресій → Phase 11b**.

Якщо після Pass 2 робив Edit'и у відповідь на 🔴 — **запусти агентів знову** (Pass 1 повторно). Регресія-фікси самі породжують нові регресії — це треба ловити у тому ж циклі, не наступної сесії.

Зупиняємось коли Pass 1 повертає «PASS — нічого критичного».
