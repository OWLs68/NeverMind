# /obsidian — Підсумок сесії для другого мозку

**Коли використовувати:** в кінці сесії. Один виклик — готовий блок для копіювання у brain сесію.

**Без аргументів.** Просто `/obsidian`.

---

## Крок 1 — Зібрати контекст сесії

1. **`git log --oneline -10`** — останні коміти
2. **`git branch --show-current`** — яка гілка
3. Переглянути контекст розмови — що обговорювали, які рішення прийняли, що відкрите

## Крок 2 — Згенерувати підсумок

Вивести **ОДНИМ код-блоком** (щоб Роман міг скопіювати кнопкою Copy):

```
Підсумок сесії [назва проекту] ([дата]):

[Що зроблено — короткий список 3-7 пунктів. Конкретно, без води.]

Ключові рішення: [якщо були]

Відкриті питання: [якщо є]

Наступний крок: [що далі]

Запиши в daily.
```

**Правила генерації:**
- Максимум 15 рядків — коротко і по суті
- Не технічний жаргон — пиши людською мовою
- Якщо сесія була про код — пиши ЩО змінилось, не які файли редагувались
- Якщо сесія була про обговорення — пиши які РІШЕННЯ прийняті
- Останній рядок завжди: "Запиши в daily."

## Крок 3 — Інструкція Роману

Після код-блоку написати:

> **Скопіюй блок вище і встав у brain сесію (roman-brain).** Claude збереже у daily файл, Obsidian підтягне при наступному pull.

## Важливо

- **НЕ** робити жодних змін у коді. Тільки генерація тексту.
- **НЕ** пушити нічого в інші репо.
- Якщо сесія була порожня — сказати "Нічого записувати — сесія без змін."
- Мова: українська.
