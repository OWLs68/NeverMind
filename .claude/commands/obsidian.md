# /obsidian — Підсумок сесії для другого мозку

**Коли використовувати:** в кінці сесії. Один виклик — готовий блок для копіювання у brain сесію.

**Без аргументів.** Просто `/obsidian`.

> **Філософія скіла (оновлено 17.04.2026 cnTkD):** brain — це **архів для майбутніх розслідувань**.
> Якщо через місяць щось зламається і треба буде зрозуміти "коли це почалось?" / "хто зробив X?" —
> daily notes мають дати відповідь. Тому підсумок **ширший за "що Роман явно обговорював"**:
> всі технічні інциденти, всі обговорення навіть якщо не прийняли рішення, всі дивні ситуації.
>
> Приклад інциденту коли це врятувало б сесії: 14.04.2026 між двох сесій Claude зникли
> деплої v54-v130 через reset без згадки у daily notes — розслідування 17.04 не знайшло автора.

---

## Крок 1 — Повний контекст сесії (ширше ніж раніше)

1. **`git log --oneline -15`** — останні коміти у поточній гілці
2. **`git log --all --oneline --since="today 00:00"`** — всі коміти сьогодні (у всіх гілках)
3. **`git branch --show-current`** — яка гілка активна
4. **`git diff main --stat`** — що відрізняється від main (якщо на feature-гілці)
5. **Перечитати розмову сесії цілком** — звернути особливу увагу на:
   - Що Роман просив зробити (explicit requests)
   - Що він ОБГОВОРЮВАВ без запиту зробити (concepts, ідеї, сумніви) — **важливо це не пропустити**
   - Які питання залишились без відповіді
   - Рішення які приймались ("вирішили що…", "відмовились від…")
   - Зміни які зробив Claude (навіть якщо Роман не явно просив)
   - Інциденти: помилки, reset, force push, збої CI, відкати, git-аномалії

## Крок 2 — Структура підсумку (РОЗШИРЕНА)

Вивести **ОДНИМ код-блоком** (щоб Роман міг скопіювати кнопкою Copy):

```
Підсумок сесії NeverMind [YYYY-MM-DD] — сесія [XXXXX]:

🔧 Що зроблено (код/файли):
- [список конкретних змін 3-10 пунктів. Якщо нічого не робили у коді — написати "Код не чіпали"]

💬 Що обговорили (навіть якщо не робили):
- [всі теми які Роман підняв: концепти, ідеї, сумніви, плани на майбутнє, скарги]
- [якщо обговорення було порожнє — "Без додаткових обговорень"]

🎯 Ключові рішення:
- [формат "вирішили X, бо Y" — якщо рішення було прийнято у цій сесії]
- [якщо рішень не було — "Без окремих рішень"]

⚠️ Інциденти (критично для майбутніх розслідувань):
- [всі `git reset`, `git push --force`, revert, видалення файлів — ОБОВ'ЯЗКОВО з hash комітів]
- [всі падіння CI / зламані деплої / баги які поламали прод / відкати до попередньої версії]
- [всі дивні ситуації які важко пояснити — навіть якщо вирішилось]
- [якщо нічого незвичайного — "Без інцидентів"]

🔓 Відкриті питання:
- [не вирішено у цій сесії, чекає наступної]
- [або "Без відкритих питань"]

➡️ Наступний крок:
- [що робити у наступній сесії]

📊 Діапазон версій: v[від] → v[до] (якщо були деплої)
🌿 Гілка: claude/start-session-[XXXXX]
🔗 Коміти: [перший hash] → [останній hash] ([скільки] комітів)

Запиши у brain.
```

**Правила генерації:**
- Середня довжина — 20-40 рядків (не "максимум 15" як раніше — Роману потрібен контекст для майбутніх пошуків)
- Кожна секція ОБОВ'ЯЗКОВА — навіть якщо вона "порожня" (`Без інцидентів` / `Без відкритих питань`). Це **сигнал для розслідування**: якщо секція "Інциденти" написана "Без інцидентів" у день X — значить Claude у цей день НЕ робив reset. Якщо її нема взагалі — невідомо
- Не технічний жаргон у тексті секцій (але hash комітів / версії / назви файлів — обов'язково точні, для `grep` у майбутньому)
- Якщо сесія була про код — пиши ЩО змінилось (фіча/баг), додай які файли торкнулись
- Якщо сесія була про обговорення — пиши які ТЕМИ підіймались, навіть якщо без рішень
- Останній рядок завжди: "Запиши у brain."

## Крок 3 — Інструкція Роману

Після код-блоку написати:

> **Скопіюй блок вище і встав у brain сесію (roman-brain).** Claude у brain-сесії сам класифікує підсумок і покладе у правильний файл (для NeverMind — `projects/nevermind/nm-YYYY-MM-DD.md`, для особистих записів — `daily/`). Obsidian підтягне при наступному pull.
>
> **Особливо звертай увагу на секцію "Інциденти":** якщо там щось є — це критично для майбутніх розслідувань. Краще мати порожню секцію "Без інцидентів" ніж взагалі пропустити блок.

## Важливо

- **НЕ** робити жодних змін у коді. Тільки генерація тексту.
- **НЕ** пушити нічого в інші репо.
- Якщо сесія була порожня — все одно виводити стандартний блок з `Код не чіпали` / `Без інцидентів` / `Без відкритих питань` і версіями — бо факт "у цей день нічого не робили" теж корисний для майбутніх розслідувань.
- Мова: українська (для тексту секцій), англійська технічна (для команд/hash/файлів).
- Інциденти **НЕ приховувати** — якщо Claude зробив щось незаплановане (навіть автоматично), це МАЄ бути у підсумку. Чесність важливіша за гарний вигляд сесії.
