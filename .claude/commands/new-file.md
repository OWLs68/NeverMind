# /new-file — Додавання нового JS модуля у `src/`

Аргумент: шлях до нового файлу в `src/`. Наприклад: `/new-file src/tabs/calendar.js` або `/new-file src/owl/animator.js`.

> **Важливо:** проект використовує **ES Modules** з бандлером `esbuild`. Нові файли йдуть у `src/` і підключаються через `import` у `src/app.js`. `sw.js` STATIC_ASSETS кешує **тільки** `bundle.js` — окремі файли у кеш не додаються. У `index.html` тільки один `<script src="bundle.js">`.

## Крок 1 — Обрати правильну папку

- `src/core/` — ядро (нав, boot, utils, trash, logger)
- `src/ai/` — AI-інтеграція (зараз тільки `core.js`)
- `src/owl/` — OWL агент (board, inbox-board, proactive, chips)
- `src/ui/` — UI helpers (keyboard, swipe-delete)
- `src/tabs/` — окрема вкладка (inbox, tasks, habits, notes, ...)

Якщо не впевнений куди покласти — запитати Романа перед створенням.

## Крок 2 — Створити файл з базовою структурою

```javascript
// Імпорти з інших модулів (приклад)
import { switchTab, showToast } from '../core/nav.js';
import { callAI, getAIContext } from '../ai/core.js';

// Експортовані функції (які використовують інші модулі)
export function myNewFunction() {
  // ...
}

// Window globals — ТІЛЬКИ якщо функцію викликає inline onclick у index.html
window.myNewFunction = myNewFunction;
```

**Правила:**
- Відносні шляхи імпортів (`../core/nav.js`), не абсолютні
- Експортувати тільки те що потрібно іншим модулям
- Локальні функції (`getTasks()`, `saveNotes()` і т.д.) — інші модулі кличуть напряму через імпорт
- Не створювати циклічних залежностей

## Крок 3 — Додати імпорт у `src/app.js`

Відкрий `src/app.js`, знайди правильне місце за порядком:
```
core/ → ai/ → owl/ → ui/ → tabs/
```

Додай `import './твій-новий-шлях.js';` у відповідну секцію. **Порядок критичний** — відповідає порядку оригінальних `<script>` тегів з часів до-рефакторингу і уникненню циклічних залежностей.

## Крок 4 — Оновити таблицю файлів у `CLAUDE.md`

Відкрий `CLAUDE.md`, знайди секцію **"Файлова структура"** (таблиця). Додай новий рядок з шляхом і коротким описом відповідальності.

## Крок 5 — Перевірка синтаксису

```bash
node --check src/шлях/файл.js
```

Має пройти без помилок.

## Крок 6 — (Опційно) Локальна збірка для перевірки

```bash
node build.js
```

Згенерує `bundle.js`. Якщо є помилки імпортів — esbuild покаже які.

> **⚠️ НЕ РОБИТИ:**
> - Не створювати файл у корені (`app-*.js` — стара структура, проект мігрував на `src/`)
> - Не додавати новий `<script>` тег у `index.html`
> - Не додавати файл у `STATIC_ASSETS` у `sw.js` — кешується тільки `bundle.js`
> - Не комітити `bundle.js` — CI генерує його під час деплою
