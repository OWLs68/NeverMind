# /new-file — Додавання нового JS файлу

Аргумент: назва файлу. Наприклад: `/new-file app-calendar.js`

## Крок 1 — Створи файл
Створи файл з базовою структурою в корені проекту.

## Крок 2 — sw.js → STATIC_ASSETS (список файлів для кешу)
Відкрий `sw.js`, знайди масив `STATIC_ASSETS` і додай новий файл до списку.
Якщо не додати — браузер не кешує файл і PWA (Progressive Web App — веб-додаток що працює офлайн) не працюватиме без інтернету.

## Крок 3 — index.html → script тег (тег підключення скрипта)
Відкрий `index.html`, знайди блок з `<script>` тегами в кінці файлу.
Додай новий тег у правильне місце — ПІСЛЯ файлів від яких залежить, ДО файлів що залежать від нього.

⚠️ Порядок `<script>` тегів критичний — скрипти залежать один від одного. Перевір залежності перед вставкою.

## Крок 4 — Оновити CLAUDE.md
Додай новий файл в таблицю "Файлова структура" в `CLAUDE.md` з описом відповідальності.

## Крок 5 — Перевірка
`node --check назва-файлу.js`
