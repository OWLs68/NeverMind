# /ux-ui — UI-код адаптований під DESIGN_SYSTEM.md

Цей скіл активується перед будь-яким UI-кодом у NeverMind (модалка, картка, компонент, стилізація). Він блокує "дефолтні" інтерфейси (білі модалки, фіолет, довільні паддінги) і змушує спочатку прочитати дизайн-систему і шаблон існуючого схожого компонента.

---

## ⚠️ Перед виконанням — обов'язкові перевірки

1. **Прочитати `docs/DESIGN_SYSTEM.md` ПОВНІСТЮ перед будь-якою UI-правкою.** Особлива увага — секція "⚠️ Помилки які були зроблені" зверху файлу. Там описано реальний інцидент з padding'ом модалок (01.04.2026) — 5 марних спроб через невірну архітектуру. Якщо не прочитав — зупинись.

2. **Знайти існуючий схожий компонент.** Перед написанням нової модалки/картки/форми — знайти аналог у коді. Приклади:
   - Модалка задачі → `src/tabs/tasks.js` (функція `openTaskModal`)
   - Модалка звички → `src/tabs/habits.js`
   - Модалка операції (glass) → `src/tabs/finance.js` → `_renderTransactionModalBody`
   - Картка задачі → `index.html` + `style.css` (клас `.task-card` або схожий)
   
   **Якщо аналог існує — копіювати його патерн, не вигадувати новий.**

3. **Заборонені кольори.** `docs/DESIGN_SYSTEM.md` → секція "❌ Заборонені кольори":
   - **Фіолетовий — НЕ використовувати** (Роман не любить). Claude за замовчуванням пропонує — це залишок з тренувальних даних.
   - Альтернативи: `#c2790a` (amber) / `#1e1040` (основний темний) / `rgba(255,255,255,0.X)` (glass).

4. **Правила padding для модалок (ЖОРСТКІ):**
   - Горизонтальний padding (`padding: 0 20px`) — **ТІЛЬКИ на outer panel** (той що має `overflow:hidden` і `border-radius`).
   - Scroll-контейнер всередині — **тільки вертикальний padding** (`padding: 28px 0 calc(env(safe-area-inset-bottom) + 28px)`).
   - Padding на `overflow-y:auto` дочірніх елементах **НЕ дає ефекту** у ряді браузерів — це реальний інцидент з 01.04.2026.

5. **Опис перед кодом (правило з CLAUDE.md).** Перед кодом UI-задачі:
   - (а) Посилайся на існуючий схожий компонент ("як модалка операції у `src/tabs/finance.js` → `_renderTransactionModalBody`").
   - (б) Опиши словами що де розміщене і як взаємодіє.
   - (в) Для зовсім нових компонентів без аналогів — виклич скіл `/mockup` (ASCII макет).
   - (г) Чекай підтвердження Романа ПЕРЕД кодом.

6. **CSS/layout — описати результат у числах перед пушем.** Приклад формулювання:
   > "outer panel 480px max-width, padding 0 20px з кожного боку → вміст буде 480−40=440px."
   
   Якщо не можу описати у числах — не пушу.

7. **CACHE_NAME bump обов'язковий** після будь-якої зміни `index.html` / `style.css` / JS-модуля (команда `date +"nm-%Y%m%d-%H%M"`).

**Якщо хоч одна перевірка знаходить конфлікт з DESIGN_SYSTEM.md — зупинись і повідом Роману перед кодом.**

## Крок 1 — Класифікувати тип UI-задачі

| Тип | Приклади | Що читати обов'язково |
|-----|----------|----------------------|
| **Модалка** | редагування задачі, форма операції, налаштування картки | `docs/DESIGN_SYSTEM.md` → секція "Модальне вікно" + існуюча модалка-аналог |
| **Картка у списку** | картка задачі, звички, транзакції | `docs/DESIGN_SYSTEM.md` → "Картки задач" + існуюча картка-аналог |
| **Кнопка / чіп** | чіп OWL, кнопка дії, tab-item | `src/owl/chips.js` (чіпи), `index.html` (tab-items), `style.css` (класи кнопок) |

## Крок 3 — Для модалки: скопіювати HTML-шаблон

Шаблон у `docs/DESIGN_SYSTEM.md` → секція "Модальне вікно" → "HTML-шаблон (актуальний)". **Не вигадувати структуру — брати готову.**

Ключові правила структури:
- Overlay: `padding: 0 16px 16px`
- Backdrop: `rgba(0,0,0,0.35) + blur(4px)`
- **Outer panel:** `overflow:hidden` + `padding: 0 20px` + `max-width: 480px`
- **Scroll container (всередині):** `overflow-y:auto` + `padding: 28px 0 calc(env(safe-area-inset-bottom) + 28px)`
- Handle риска: 36×4px, `rgba(0,0,0,0.12)`

## Крок 4 — Свайп закриття для модалки

```javascript
// Імпорт з src/tabs/tasks.js:
import { setupModalSwipeClose } from '../tabs/tasks.js';

setupModalSwipeClose(
  document.querySelector('#your-modal > div:last-child'),
  closeYourModal
);
```

Функція `setupModalSwipeClose` має **guard від drum-col** (колонки барабанних пікерів) — не ламає їх свайп.

## Крок 5 — Опис результату у повідомленні Роману

Перед пушем написати Роману у такому форматі:
> "Зробив [що]. Outer panel [X]px × max-height [Y]vh з padding 0 20px — вміст [X−40]px. Scroll container з padding 28px зверху і calc(safe-area + 28px) знизу. Колір кнопки Зберегти: linear-gradient(135deg, [hex1], [hex2])."

Це дає Роману розуміння що саме пушиться. Якщо опис не вмію скласти — код ще не готовий.

## Важливо

- **НЕ** переробляти існуючі модалки "бо там щось некрасиво" — спочатку узгодь з Романом. Якщо зміна в дизайн-системі — оновити `docs/DESIGN_SYSTEM.md`.
- **НЕ** створювати нові окремі CSS-файли. Весь CSS — у `style.css`.
- **НЕ** розбивати `index.html` на кілька файлів. PWA на iOS краще працює з одним HTML (жорсткий закон з CLAUDE.md).
- Якщо модалка має **свайп закриття** — обов'язково `setupModalSwipeClose`. Без нього модалки закриваються тільки тапом по backdrop або кнопкою.
- Для **input/textarea в glass-модалках** — фон `rgba(255,255,255,0.7)` (інакше не видно на 30% glass).
