Дві причини: (а) `INSIGHTS_VERSION=2` закешував старий звіт що міг бути згенерований до того як юзер відмітив звички; (б) `getAIContext()` давав звичкам тільки сьогодні (`toDateString`) — будь-який чат/табло на «3 дні не ходив у зал» галюцинував. У `getAIContext` після «Звички сьогодні» додано блок `[РЕАЛЬНІ ДАНІ ЗА 7 ДНІВ]`: для кожної build-звички `done/scheduled (pct%)` за вікно 7 днів (логіка з `_buildWindowContext` з `me.js`), quit-звички (`abstained/7 днів утримання`), закриті задачі за тиждень. Поширюється на всі 8 чатів через спільний `getOWLPersonality()`.

4. **«Відкрий звички» з чату Я → реально підтаб Звички + блок «OWL знає тебе» видно** (`6bc5d5b`) — два баги lRnXU за один коміт. Фікс: фон білий 0.85 + рамка темніша 0.35 + м'яка тінь.

5. **Видалено онбординг-модалку при першому вході** (`1049c0f`) — Роман прямо сказав: «Видали онбординг при вході там де писати імʼя і АРІ. Залиш ці блоки в налаштуваннях».   ```js
  function runMigrations() {
    const currentVer = parseInt(localStorage.getItem('nm_db_version') || '0');
    const targetVer = 1;
    if (currentVer >= targetVer) return;
    const backup = JSON.stringify(localStorage);
    localStorage.setItem(`nm_backup_v${currentVer}`, backup);
    try {
      if (currentVer < 1) {
        let tasks = JSON.parse(localStorage.getItem('nm_tasks') || '[]');
        tasks = tasks.map(t => ({ ...t, id: t.id || crypto.randomUUID() }));
        localStorage.setItem('nm_tasks', JSON.stringify(tasks));
      }
      localStorage.setItem('nm_db_version', targetVer.toString());
    } catch (error) {
      console.error('Migration failed!', error);
    }
  }

## 🛡 Safari/Chrome zombie-SW (B-79) — історична довідка

**Симптом:** юзер бачив у Safari/Chrome стару версію (v53 · 07:58 17.04) попри PWA v199. Різниця ~150 деплоїв.

**Root cause:** до B-73 фіксу (07:45 UTC 17.04) SW був cache-first для ВСІХ файлів включно з `sw.js`. iOS Safari перехоплював навіть update check через SW fetch listener (всупереч W3C spec). Старий SW сам себе увічнював.

**Фікс для застрягших (ручний, один раз):**
- Safari iOS: Налаштування → Safari → Додатково → Дані сайтів → `owls68.github.io` → Видалити
- Chrome iOS: Очистити дані перегляду для сайту

Після цього новий SW network-first (B-73) оновлюється автоматично назавжди. Meta-теги Cache-Control у `<head>` (B-79) — запобіжник на майбутнє.
