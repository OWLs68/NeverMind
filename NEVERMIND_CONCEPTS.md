## 💰 Фінанси — концепція

**Ідея:** Не трекер витрат — персональний фінансовий тренер. Фрейм: не "скільки витратив" а "скільки зберіг і що це означає".

**Структура вкладки (зверху вниз):**
1. OWL табло — проактивні інсайти з числами
2. Перемикач: Тиждень / Місяць / 3 місяці
3. Hero-карта: % заощаджень від доходу + дохід/витрати/залишок + тренд
4. Три інсайт-картки: % зміни витрат, % заощаджень, середній день vs норма
5. Прогноз до зарплати (3 сценарії)
6. Коуч-блок OWL: одна конкретна порада з числами
7. Графік витрат і доходів по тижнях
8. Категорії з кольоровим статусом (🟢🟡🔴) + benchmark 50/30/20
9. Останні транзакції

**Налаштування Фінансів** (у вкладці, НЕ в загальних): бюджет, дата зарплати, ліміти, очищення, експорт.

**Підкатегорії:**
- Їжа → Продукти, Ресторан, Кафе, Доставка, Фастфуд
- Транспорт → Паливо, Таксі, Парковка, Громадський, Ремонт авто
- Підписки → Стрімінг, Музика, Хмара, Додатки, Ігри
- Здоров'я → Аптека, Лікар, Спортзал, Аналізи, Косметика
- Житло → Оренда, Комунальні, Інтернет, Ремонт, Меблі
- Покупки → Одяг, Техніка, Книги, Подарунки, Дім

**База знань OWL:** Кійосакі, Рамзі, Бах (Latte Factor), Клазон (10%), Хаусел, Талеб, Сеті ("витрачай без провини на те що любиш").

## 🌙 Вечір — концепція

**Ідея:** Закриття дня. Відкрив — вже бачиш результат. Без кнопок.

**Принцип:** OWL генерує підсумок у фоні після 18:00. Використовує: закриті задачі, виконані звички, записи Inbox, моменти. Якщо відкрив вдень — прогрес на зараз.

**Структура:**
1. Кільце продуктивності дня (% виконання)
2. Підсумок від OWL (що зроблено + один інсайт + згадка про проекти)
3. Настрій дня — 5 емодзі, один тап, повністю опціонально
4. Моменти дня — кнопка "+ момент", не нав'язується
5. Фінанси сьогодні — якщо були транзакції

## 👤 Я — концепція

**Ідея:** Дзеркало розвитку. Не щоденна вкладка — тижнева. OWL знаходить патерни сам.

**Структура:**
1. Огляд тижня — одне речення від OWL + візуальна шкала
2. Теплова карта активності (14 або 30 днів)
3. Патерни від OWL (генеруються автоматично раз на тиждень)
4. Прогрес по активних проектах
5. Статистика звичок (тільки місячний огляд)
6. Аналіз OWL (глибший, раз на тиждень або за запитом)

**Відмінність від Вечора:** Вечір = сьогодні. Я = тиждень і місяць. Жодного дублювання.

## 📁 Проекти — концепція

**Ідея:** Не Trello — особистий бізнес-партнер від ідеї до результату.

**Flow:**
1. Вхід через Inbox ("хочу відкрити хімчистку") → OWL пропонує створити проект
2. Глибоке інтервью (4 питання) + використовує дані з пам'яті агента
3. Карта проекту: 3 наступних кроки + неочевидні труднощі + персональні ризики + ресурси
4. Живий супровід: нагадує, помічає стагнацію, допомагає при блокуванні
5. Аналітика: прогрес %, темп, витрати проекту, патерни активності

**Інтеграція:** Задачі ↔ Продуктивність, Витрати → Фінанси, Нотатки прикріплюються до проекту, Вечір згадує прогрес, Я показує в тижневому огляді.

## 🎮 Геймфікація — концепція

**Принцип:** Геймфікація як фон, не головна фіча. Мотивує реальні досягнення, не клікабельність.

**Маленькі бали (щодня):**
- Inbox запис +1 · Звичка +2 · Задача +3 · Момент +1 · Транзакція +1 · Вечір +1

**Великі бали (справжні досягнення):**
- Стрік 7 днів +25 · Стрік 30 днів +100 · Закритий проект +150
- Заощаджено 20%+ +50 · Всі звички за день +10 · 5 задач за тиждень +20
- Повний місяць активності +75

**Де видно:** Вкладка Я (рівень + бали + нагороди). OWL табло — органічна згадка.

**Чого НЕ робити:** Push-нотифікації, "ти відстаєш", феєрверки >2 сек, рівень на головному екрані, штраф за пропуск.

**Перед імплементацією обговорити:** назви рівнів, точні числа балів, чи є соціальний елемент (тільки після Supabase).

## 🧩 Модульний NeverMind — концепція

**Ідея:** Базове ядро + модулі які вмикаються під себе. Барабан показує тільки активні.

**Базове ядро (завжди):** Inbox · Продуктивність · Нотатки

**Модулі:** Фінанси · Проекти · Я · Вечір · Здоров'я · Харчування · Лікар · Освіта · Люди · Медитація · Бізнес

**Зв'язок з Jarvis:** Кожен новий модуль = нова область де OWL стає експертом. Достатньо модулів = OWL бачить всю картину і стає справжнім агентом.

## 📅 Календар — концепція (затверджено 30.03.2026)

**Розміщення:** кнопка 📅 між лічильниками Задачі і Звички у вкладці Продуктивність. Календар з вкладки Я — прибрати (лишити тільки кільця активності).

**UI флоу:**
1. Тап 📅 → модальне вікно з місячним календарем
2. Дні з задачами/подіями мають кольоровий маркер (🔴 критичний, 🟠 важливий, звичайний)
3. Тап на день → zoom-анімація: клітинка розширюється, дата заповнює екран → fade → список записів дня
4. У списку дня: задачі + події, кнопка `+ Додати`, редагування існуючих

**Типи записів:**
- Задача з дедлайном (`dueDate` на існуючій задачі)
- Подія — без чекбоксу, просто дата/час + нотатка (`nm_events`, новий ключ)

**Event структура:**
```javascript
{ id, title, date, time?, priority: 'normal'|'important'|'critical', notes? }
```

**Розширення Task:**
```javascript
{ ...існуюче..., dueDate?: timestamp, dueTime?: 'HH:MM', priority?: 'normal'|'important'|'critical' }
```

**Нагадування — система пріоритетів:**

| Пріоритет | Коли | Механізм |
|-----------|------|----------|
| 🔴 Критичний | За 3 дні + за день + в день | Червона картка закріплена ЗВЕРХУ Inbox |
| 🟠 Важливий | За день + в день | Помаранчева картка зверху Inbox |
| 🟡 Звичайний | В день | OWL board згадує проактивно |

Картку можна закрити свайпом. Поп-апів немає — закріплена картка в Inbox достатньо помітна і не лякає.

**getAIContext() — новий блок:**
```
Заплановано на найближчі 7 днів:
- [ID:xxx] Постірати одяг — 31 бер.
- [EVENT] Платіж за паркінг — 1 квіт. 🔴 критичний
```

**Порядок реалізації:**
1. `dueDate` + `priority` в Task. Кнопка 📅 між лічильниками.
2. Calendar modal + zoom-анімація. Маркери на днях.
3. `nm_events` — нові події. Форма додавання.
4. Закріплені картки-нагадування в Inbox.
5. `getAIContext()` + OWL нагадування по `dueDate`.

## 🎤 Голосовий ввід — концепція (затверджено 30.03.2026)

**Де:** кнопка мікрофона в полі вводу (іконка вже є в UI всіх барів).

**Технологія:** Web Speech API (`SpeechRecognition`) — вбудовано в Safari iOS 14.5+ і Chrome, безкоштовно, підтримує `lang: 'uk-UA'`.

**Флоу:**
1. Тап на мікрофон → запис починається (кнопка підсвічується)
2. Текст з'являється в полі в реальному часі
3. Пауза → зупиняється автоматично
4. Можна відредагувати або одразу надіслати

> ✅ Незалежна фіча, 1 сесія. Робити ДО Supabase.

## 🦉 Живий OWL — концепція (обговорення 30.03.2026)

**Проблема зараз:** OWL board виглядає як "стрічка новин біржі". Агент пасивний — тільки відповідає, не ініціює.

**Нова концепція — три ролі OWL:**

1. **Допитливий** — сам коментує записи користувача
   - "Бачу ти додав задачу 3 дні тому. Вже зробив?"
   - "Що таке 'автошоу'? Це подія? Хочеш нагадування?"

2. **Вчитель** — показує як користуватись додатком
   - "Спробуй написати 'купити молоко завтра' — я створю задачу з датою"
   - "Знаєш що свайп вліво видаляє запис?"

3. **Ініціатор** — починає розмову першим
   - Ранок: "Доброго ранку! Що сьогодні найважливіше?"
   - Після стріку: "5 днів поспіль! Що допомагає тримати стрік?"

**Візуально:** аватар OWL + бульбашка-повідомлення, анімація "друкує...", розмовний тон.

**Inbox:** окреме обговорення (картки vs чат-фокус).

> ⚠️ Великий редизайн. Окрема сесія планування перед імплементацією.
