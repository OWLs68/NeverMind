## `/owl-motion` — Оживлення сови

**Суть:** Claude отримує чіткі правила як малювати і анімувати OWL-персонажа (сову).

**Що робить скіл:**
- Генерує SVG-структуру сови з CSS-класами для кожного стану
- Пише CSS keyframes (кадри анімації) для кожного настрою:
  - `idle` — тихе левітування коли нема повідомлення
  - `alert` — кивок/поворот голови коли з'явилось нове повідомлення
  - `thinking` — анімація очей поки AI думає
  - `error` — опущені крила при помилці
  - `greeting` — змах крилом при вітанні
- Пише JS-клас `OwlController` який перемикає стани залежно від подій застосунку
- Анімація розгортання бабла: табло з'являється → бабл "розкривається" вправо від сови, зникає → згортається назад

**Технічне рішення:** SVG + CSS keyframes (без Lottie — важкий, без SMIL — застарілий). Працює на iOS Safari без додаткових бібліотек.

**Статус:** ⏳ Чекає на SVG-персонажа (малює не Claude — інша нейромережа або Роман)

## `/pwa-ios-fix` — Стабільність на iPhone

**Суть:** Чеклист обов'язкових перевірок і виправлень специфічних глюків iOS Safari при кожній зміні.

**Що перевіряє/виправляє скіл:**
- **Висота екрану** — iOS рахує `100vh` неправильно коли вилазить клавіатура. Фікс: CSS-змінна `--vh` яка перераховується при кожній зміні розміру вікна
- **Повернення з фону** — iPhone "засинає" і при поверненні може показати старі дані. Фікс: обробник `pageshow` з перевіркою `event.persisted` — примусово оновлює стан OWL і дані
- **Скрол** — заблокувати "гумовий" bounce (відскок) для основного body, але залишити для вкладок
- **Дотики** — `touch-action: manipulation` на всіх кнопках щоб прибрати затримку 300мс при тапі
- **Перевірка існуючих фіксів** — скіл спочатку дивиться що вже є в `src/core/boot.js` і доповнює тільки те чого нема

**Статус:** ⏳ Не реалізовано (частина фіксів вже є в `boot.js` вручну — скіл систематизує)
