У `getOWLPersonality()` між «ПРАВИЛО ЧЕСНОСТІ» і «G12» додано «ПРАВИЛО ЗОВНІШНІХ ФАКТІВ»: AI не має інтернет-доступу, на питання про фільми/книги/осіб/новини чесно каже «не знаю», реагує на враження юзера, а не вигадує сюжет. Поширюється на всі 8 чатів через спільний `getOWLPersonality()`.

4. **«Відкрий звички» з чату Я → реально підтаб Звички + блок «OWL знає тебе» видно** (`6bc5d5b`) — два баги lRnXU за один коміт. (1) `switch_tab(target:'habits')` викликав лише `switchTab('tasks')` без `switchProdTab('habits')` — юзер переходив на Продуктивність але бачив Задачі замість Звичок. Фікс: `setTimeout(() => switchProdTab('habits'), 50)` після `switchTab`. Аналогічно для `target='tasks'` явно ставимо підтаб 'tasks'. (2) Блок «🦉 OWL знає тебе» (HTML `me-weekly-insights`) мав прозорий градієнт `rgba(124,74,42,0.08→0.03)` і слабку рамку 0.18 — на бежево-кремовому фоні `#e8d5c4` сторінки «Я» візуально невидимий. Фікс: фон білий 0.85 + рамка темніша 0.35 + м'яка тінь.

5. **Видалено онбординг-модалку при першому вході** (`1049c0f`) — Роман прямо сказав: «Видали онбординг при вході там де писати імʼя і АРІ. Залиш ці блоки в налаштуваннях». Поля доступні у Налаштуваннях — окрема модалка зайва. Фікс: `checkOnboarding()` більше не показує модалку, замість цього один раз ставить `nm_onboarding_done='1'` для старих юзерів. HTML `#onboarding` лишився `display:none` за замовчуванням. `showFirstVisitTip` (toast-підказки при першому вході на вкладку) — окрема функція, не торкав.

### Обговорено (без виконання)

- **Brain-фідбек з TdIqO (6 пунктів)** — Роман відхилив усі 6: «Нічого з цього». Підтвердження що нове правило обробки brain-фідбеку (введене у lRnXU) працює — обговорили, raw НЕ зберігаємо. На відміну від попередньої сесії де я був додав raw до SESSION_STATE і Роман зупинив.
- **Плашка mood під бубликами «Я»** — обговорили варіанти (а) лишити без плашки, (б) маленький текст без рамки/тіні, (в) клікабельна плашка яка відкриває Вечір. Роман відхилив повністю: «Без плашки. Користувач сам знає свій настрій сьогодні. А от для аналізу для агента то можна лишити в Вечір». `nm_evening_mood` лишається для AI-промптів без візуалу.
- **AI плутає звички з задачами у weekly insights звіті** — після H0DxS фіксів звіт показав «Ти закрив 3 задачі за тиждень» при реальних 0 закритих задачах і 3 виконаних звичках сьогодні. Корінь: промпт у `generateWeeklyInsights` недостатньо чітко розрізнює сутності (AI взяв число «3» з виконаних звичок і назвав їх задачами). Фікс пропонувався — Роман викликав `/finish` без «Роби». Лишилось на наступну сесію.

### Ключові рішення

- **Корінь «OWL знає тебе бреше» = ДВА місця: кеш + контекст.** Фіксимо обидва за один прохід (bump VERSION + новий блок у getAIContext) — щоб не повертатись.
- **Правило ЗОВНІШНІХ ФАКТІВ — у universal-блоці `getOWLPersonality()`** (для всіх 8 чатів одразу), не у конкретному `evening-chat` промпті. Один мозок = одне правило.
