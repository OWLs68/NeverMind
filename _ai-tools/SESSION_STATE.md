### Блок Б — Фаза 1.5 (синк картка ↔ календар)

### Блок Б — Фаза 1.5 (синк картка ↔ календар)

**Архітектура:** "lazy housekeeping" (ліниве прибирання) замість push-нотифікацій між модулями. Це усуває circular import (`health` ↔ `calendar`) — calendar.js повністю незалежний від health.js, синк відбувається при кожному відкритті вкладки Здоров'я.

**Точки інтеграції:**
- **Картка → подія** (eager — одразу): `_syncCardAppointmentToEvent()` у `health.js` — викликається з `saveHealthCardFromModal`. Створює/оновлює подію у `nm_events` з `title: "Прийом: {назва}"`, `priority: 'important'`, `sourceCardId`, `archived: false`. Зберігає `eventId` у `card.nextAppointment.eventId`.
- **Видалення картки → видалення події** (eager): `deleteHealthCardFromModal` додає прив'язану подію у `addToTrash('event', ...)` (відновлюється 7 днів).
- **Архівація минулих** (lazy у `renderHealth`): `_archivePastAppointments()` — якщо `nextAppointment.date < today`: подія `archived:true` (лишається у Календарі як історія) + у картці `history` запис `doctor_visit` ("Прийом відбувся {date} о {time}") + `nextAppointment = null`.
- **Зворотний синк дат** (lazy): `_syncEventDatesToCards()` — якщо подія була змінена у Календарі (нова дата/час), оновлює `nextAppointment.date/time`.
- **Orphan-detection** (lazy): `_detectOrphanAppointments()` — якщо `eventId` вказує на видалену подію, прибирає `eventId` (картка лишається з `nextAppointment` без прив'язки, при наступному save створиться нова подія).
- **Реактивація архівованої:** при виставленні нової майбутньої дати у вже archived подію — `archived:false` повертається автоматично у `_syncCardAppointmentToEvent`.

**Структура події розширена** (опціональні поля): `sourceCardId` (id картки-джерела), `archived` (boolean — для минулих прийомів-документів). Існуючі події не зачіпаються — нові поля просто `undefined`.

**Структура `nextAppointment` розширена:** `{ date, time, eventId? }`.

**Edge cases НЕ реалізовано** (свідомий вибір — не блокатор для MVP, перенесено у `💡 Ideas`):
- Конфлікти часу з існуючими подіями (попередження "Перенести?")
