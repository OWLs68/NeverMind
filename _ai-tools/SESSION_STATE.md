### B-16 — Централізація системи чіпів (зроблено)

**`src/owl/chips.js` — повністю переписаний як центральний модуль:**
- `renderChips(containerEl, chips, tab, options)` — єдина функція рендеру чіпів (була дублікат в `board.js` + `inbox-board.js`)
- `CHIP_PROMPT_RULES` — єдиний текст правил для AI промптів (був скопійований у 3 файли: `proactive.js`, `inbox-board.js`, `ai/core.js`)
- `CHIP_JSON_FORMAT` — формат JSON відповіді AI
- `handleCompletionChip(text)` — ✔️ чіпи закривають задачі/звички ЛОКАЛЬНО через fuzzy match (нечіткий пошук по 4-літерних стемах), без виклику AI
- `normalizeChips(chips)` — backward compat для старих string-чіпів
- `handleChipClick(tab, text, action, target)` — головна логіка обробки кліку
- `sendChipToChat(tab, text)` — відправка чат-чіпа у відповідну вкладку
- Чіпи зникають після кліку з анімацією (opacity + scale)

**Промпт переписаний:** чіпи = варіанти швидкої ВІДПОВІДІ користувача, НЕ заклики до дії.

## 🔴 Що треба зробити далі

### B-16 (незакрита частина): чіпи-привиди на інших вкладках

**Проблема:** коли юзер закриває задачу вручну (через галочку в Продуктивності), чіп з ✔️ для цієї задачі залишається на ІНШИХ вкладках (Нотатки, Я, тощо). Текст табло теж не оновлюється ("купи м'ясо" хоча вже куплено).

**Рішення:** в `renderChips()` додати фільтр — перед малюванням кожного ✔️ чіпа перевіряти чи задача/звичка ще активна. Якщо вже закрита → не показувати. Потрібно:
1. Витягти логіку fuzzy match з `handleCompletionChip()` в окрему функцію `isCompletionDone(text)` — тільки перевіряє, не закриває
2. Використати її в `renderChips()` для фільтрації перед рендером
3. Це має працювати для ВСІХ вкладок одночасно — вкладки повинні знати що відбувається між ними

### B-17: табло не оновлюється миттєво

**Проблема:** табло реагує тільки на 3-хвилинний таймер (Inbox) і переключення вкладок.

**Рішення:** debounce-функція `requestBoardUpdate(tab)` — при ключових подіях (закриття задачі, виконання звички, нова нотатка, витрата, момент) чекає 3 сек і запускає нову генерацію повідомлення табло.
