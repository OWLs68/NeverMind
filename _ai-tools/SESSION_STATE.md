### Обговорено (без виконання)

- **Prompt cache TTL у Claude Code:** не налаштовується user-side. Стандарт 5 хв зашитий у CLI. 1-годинний кеш (дорожче у ~2 рази) — тільки через API напряму. Практичний висновок: не робити довгих пауз щоб не платити за перерахунок промпту.

- **iOS застосунок vs комп для хуків:** не має значення. Claude Code крутиться у контейнері на серверах Anthropic, хуки живуть у `.claude/` репо → працюють після `git pull` незалежно від клієнта.

### Ключові рішення

- **Поріг попередження 800K (80%)** — щоб мати запас ~20 повідомлень до auto-compaction.
- **Апроксимація через байти transcript** — компроміс між простотою (bash) і точністю. Коли Роман перевірить `/context` у реальному використанні — можна буде підкрутити `BYTES_PER_TOKEN`.
- **Хук додається до існуючого UserPromptSubmit масиву** — Claude Code підтримує кілька команд у одному хук-event, виконуються послідовно.
