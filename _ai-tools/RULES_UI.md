# RULES_UI — UI/UX задачі

> Читай при будь-якій задачі що зачіпає вигляд: модалки, кольори, анімації, свайпи, кнопки, шрифти.
>
> **Дизайн-токени, спеки модалок, anti-patterns** → `docs/DESIGN_SYSTEM.md` (932 рядки, повна довідка).

---

## 1. UI — опиши перед кодом

Перед кодом UI-задачі:

1. **Посилайся на існуючий схожий компонент** — наприклад «як модалка звички у `src/tabs/habits.js`», «як кнопка Зберегти у Задачах», «як смужка-ручка зверху у Звичках».
2. **Опиши словами** що де розміщене і як взаємодіє: «Внизу екрану з'явиться кнопка Готово. Тап → закриває модалку. При свайпі вниз — модалка їде за пальцем».
3. **Для зовсім нових компонентів без аналогів** — використай скіл `/mockup` (ASCII макет). Чекай підтвердження від Романа перед кодом.

**Чекай ок від Романа.** Без підтвердження — не пиши CSS/HTML, навіть якщо здається тривіальним.

---

## 2. Описуй ВИГЛЯД, не назви коду

Роман не дивиться у код — він дивиться на екран телефону. Тому пояснюй UI **тим що він бачить**, не назвами CSS-властивостей.

### ❌ Так НЕ пояснюй (реальна помилка з 17.04.2026)

> «Знайшов корінь у `src/core/boot.js:216-232` (функція `setupSettingsSwipe`). Три дефекти: `startScrollTop === 0` обмежує закриття коли scrollTop не 0. Немає `touchmove` з `translateY` — нема візуального фідбеку. Конфлікт з iOS системним свайпом зверху — панель висотою 88vh.»

> «Іконки Мова/Сповіщення — stroke `#4f46e5` (indigo-600), фон `rgba(129,140,248,0.18)`. OWL active border `#7c3aed` (violet-600). Треба замінити на `#c2790a` (amber).»

### ✅ Так ТРЕБА

> Свайп закриття Налаштувань поганий з 3 причин:
> 1. Працює тільки коли екран у самому верху. Прокрутив донизу — свайп не закриває.
> 2. Поки тягнеш пальцем — нічого не рухається. На задачах модалка їде за пальцем, тут стоїть. Здається що зламано.
> 3. iPhone перехоплює свайп зверху — думає що ти хочеш відкрити панель сповіщень.
>
> Зроблю як на задачах — там працює.

> На Налаштуваннях є 3 фіолеті штуки (а ти фіолет не любиш):
> 1. Іконки «Мова» і «Сповіщення» — на світло-фіолетовому фоні
> 2. Вибрана OWL-картка обводиться фіолетом
> 3. Кнопки € / УКР / «Зберегти» — темно-фіолетові
>
> Поміняю на бурштиновий + помаранчевий (як всюди для вибраного).

## 3. Правила формулювання UI-пояснень

- ❌ Не називати функції, CSS-класи, HEX-коди, файли, рядки коду в основному поясненні
- ❌ Не використовувати `overflow-y`, `translateY`, `backdrop`, `outer panel`, `stroke`, `primary`, `indigo`, `violet-600`
- ✅ Описувати вигляд словами: «фіолетовий фон», «темна кнопка», «їде за пальцем», «перекреслюється»
- ✅ Посилатись на інші місця застосунку як приклад: «як кнопка Зберегти у Задачах», «як смужка-ручка зверху у Звичках»
- ✅ Пояснювати наслідок для юзера: «не закриється поки не прокрутиш до верху» краще ніж «`scrollTop !== 0` блокує»
- ✅ Якщо ТРЕБА згадати файл/функцію (наприклад при комітах) — тільки в самому кінці, окремо, не в поясненні проблеми

### «Один спосіб пояснити»

Не дублюй технічну і людську версію в одному повідомленні. Обирай людську. Якщо Роман попросить деталі — тоді покажи технічну.

## 4. CSS/layout — описати результат перед пушем

Приклад правильного формулювання:

> «Зовнішній блок (той що видно зсередини) 360px, внутрішній відступ 20px з кожного боку → вміст буде 360−40 = 320px.»

Це дає Роману розуміння що саме ти пушиш на продакшен. Якщо не можу описати — не пушу.

## 5. iOS Safari візуальний баг — обов'язковий чек ДО CSS-патчів

**Тригер:** симптом на iPhone «модалка глючить / стискається / мерехтить / обрізається / стрибає / реагує на тап». Перш ніж писати CSS-патч — пройти 3 точки. 30 секунд роботи замість 4 ітерацій false leads.

**Чек:**

1. **`grep ":active\|:focus\|:hover" style.css`** на universal selectors (`button`, `[onclick]`, `*`). Глобальне правило `[onclick]:active { transform: scale(0.87); }` чіпляється до **всіх** елементів з onclick — включно з root-модалками. Якщо знайшов — корінь скоріше за все тут.

2. **`grep "backdrop-filter" style.css`** + перевірка чи символ скаржиться має `backdrop-filter` parent. iOS Safari nested blur layers (parent + child) при momentum scroll re-rasterize → subpixel rounding glitch / clipping. Фікс — або `translateZ(0)` на дитині (ізолює composite layer), або solid fill замість translucent.

3. **Composite layers + parent transform.** Якщо елемент обрізається — перевір чи parent має `transform`/`backdrop-filter`/`filter`. Це створює новий containing block який клипає absolutely-positioned дітей. Виносити overlay як sibling top-level (calendar-pattern), не дитину.

**Тільки після цих 3 кроків** — починай гіпотези CSS-патчів (mask-image, flex layout, body-lock, padding).

**Підтверджені кейси (UvEHE 03.05):** Settings scale-glitch — 4 ітерації false leads (mask, flex, nested blur, body-lock). Корінь у пункті 1: глобальне `[onclick]:active`. Chips clipping — теж 4 ітерації (mask, padding, flex-shrink, scrollIntoView). Корінь у пункті 2: parent `.ai-bar-chat-window` з `backdrop-filter:blur(16px)` клипав chips, фікс `translateZ(0)`. Обидва знайдені Council `code-regression-finder` — порівняння робочого vs зламаного компоненту.

## 6. Священні корови UI (з `docs/DO_NOT_TOUCH.md`)

- **Чат-бари на ВСІХ вкладках обов'язкові.** Не прибирати ні з одної вкладки.
- **Глобальна анімація натискання у `style.css`** — не розширювати селектор без обговорення.
- **`attachSwipeDelete`** — базова функція для свайп-видалення. Не писати свій свайп для нових списків.
- **Фіолетовий — заборонено** (Роман не любить). Використовувати бурштиновий/помаранчевий для виділення.
- **Padding модалок** — після інциденту 01.04 не перевіряти на око. Описувати математично перед пушем.
