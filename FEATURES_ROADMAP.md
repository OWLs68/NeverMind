### 📋 Що в планах

#### Фаза 1: Архітектура мозку (фундамент)

**1.1 Об'єднання табло Inbox і табло вкладок**
- Зараз: дві окремих функції генерації (`generateOwlBoardMessage` в `inbox-board.js` + `generateTabBoardMessage` в `proactive.js`), два debounce-listeners, два cooldown-механізми, дві логіки тригерів
- Треба: одна функція `generateBoardMessage(tab)` для БУДЬ-якої вкладки включно з inbox. Один debounce, один cooldown, один контекст. Різниця тільки в даних які збираються для кожної вкладки
- Результат: менше коду, менше дублювання, простіше додавати нові фічі

**1.2 Синхронізація табло ↔ чат**
- Зараз: табло і чат — два окремих потоки які не знають одне про одного
- Треба: табло бачить останні повідомлення чату (щоб не повторювати те що вже обговорили). Чат пам'ятає що табло казало (щоб відповіді були послідовні)
- Технічно: при генерації повідомлення табло додавати останні 3 повідомлення з `nm_chat_{tab}` в контекст

**1.3 Єдиний AI-обробник для всіх чатів**
- Зараз: 8 окремих функцій відправки (`sendToAI`, `sendTasksBarMessage`, `sendNotesBarMessage`...) з вузькими промптами
- Треба: всі чат-бари на всіх вкладках використовують ту саму AI-логіку з повним `getAIContext()` і повним набором actions (save, save_finance, complete_habit, complete_task тощо). Написав "витратив 200 на обід" на вкладці Продуктивність → OWL зберіг у фінанси, не перекидаючи на іншу вкладку
- Принцип: мінімальне тертя — пиши де стоїш, OWL розбереться сам
- Чати залишаються окремими для юзера (кожна вкладка = своя історія), але мозок бачить ВСІ чати при генерації відповідей

#### Фаза 2: Awareness (обізнаність мозку)

**2.1 Welcome Back (привітання при поверненні)**
- Зберігати `nm_last_active` (timestamp останнього візиту) при `visibilitychange` (подія браузера "вкладка стала невидимою")
- При поверненні перевірити: скільки часу не було, що пропущено (звички, дедлайни), чи змінилась фаза дня
- Згенерувати персоналізоване привітання: "Ого, насичений ранок? Звички поки пусті, але ще є час наздогнати"
- Складність: мала (~30 рядків)

**2.2 Крос-контекст (зв'язок між вкладками)**
- Зараз: OWL на вкладці Задачі бачить тільки задачі
- Треба: міні-лог останніх дій `nm_recent_actions` (масив, макс 20 записів). Кожна значуща дія записується: `{action: 'complete_task', title: 'Купити продукти', tab: 'tasks', ts: ...}`
- OWL на будь-якій вкладці бачить що відбувалось на інших. Записав момент "жахливий день" → OWL на Задачах каже "день непростий, може відкласти неважливе?" замість "5 незакритих задач!"
- Інтеграція: в `getTabBoardContext()` додавати останні 2-3 дії з інших вкладок

#### Фаза 3: Навчання і пам'ять

**3.1 Трекінг чіпів (навчання через кнопки)**
- `nm_chip_stats` — які чіпи натискають, які ігнорують
- Натиснув чіп → записати label і action. Чіпи зникли без кліку (нова генерація перезаписала) → ignored +1
- Передавати в промпт: "Користувач часто натискає ✔️-чіпи, рідко натискає 'Розкажи більше'" → OWL вчиться пропонувати те що працює
- Складність: мала

**3.2 nm_insights (факти про користувача)**

- Складність: середня (потребує UI)

#### Фаза 4: Jarvis Architecture (обговорено 07.04.2026 з Gemini + Claude)

> Візія: OWL = персональний Jarvis. Не чат-бот а Reasoning Engine (мозок що думає). Думає у фоні, записує факти без команди, розуміє контекст. Основна частина — ПІСЛЯ Supabase.

**ДО Supabase (можна зараз):**

| # | Назва | Суть | Складність |
|---|-------|------|------------|
| 4.1 | **Tool Calling** | Перевести API з текстового JSON на OpenAI function calling. AI сам вирішує які інструменти викликати і скільки (multi-tool за один запит). "Завтра суд" → AI одночасно: create_event + save_memory + відповідь текстом | Середня (2 сесії) |
| 4.2 | **Day State (наративна пам'ять дня)** | Середній шар пам'яті між чатом і nm_memory. Кожні 2 години або 10 повідомлень → AI генерує 2 речення: "Роман зранку закрив декларацію, після обіду обговорив звички, ввечері настрій покращився". `nm_day_narrative`. Передається в промпт табло ЗАМІСТЬ 30 сирих повідомлень чату | Мала (1 сесія) |
| 4.3 | **Семантичні cooldowns** | AI повертає `"topic": "daily_mood"` разом з текстом табло. Коли юзер відповів — тема в cooldown на 3 години. Промпт забороняє ТЕМИ а не слова. Видалити _isTooSimilar і _extractBannedWords | Мала (1 сесія) |
| 4.4 | **Relevance Scoring (локальний)** | Бали доречності (0-10) без API: повернувся після 30 хв = +3, дедлайн сьогодні = +3, чат відкритий = -3, ігнорував 3 табло = -5. Якщо бал < 5 → НЕ генерувати табло. Прибирає "набридливу муху" | Мала (1 сесія) |
| 4.5 | **Блокування табло поки чат відкритий** | Не генерувати табло поки activeChatBar відкритий + 10 сек після закриття | Тривіальна |
| 4.6 | **Contextual Personality Shifts** | OWL адаптує не тільки тон а й поведінку: зранку = план, в потоці = мовчить, після невдачі = підтримка, після перемоги = святкує. getContextMode(): planning/flow/struggling/celebrating/reflecting/returning | Мала (1 сесія) |
| 4.7 | **Мовчазний запис** | Через Tool Calling AI сам вирішує коли save_memory(). Юзер закрив "Стоматолог" → OWL мовчки: "Був у стоматолога 07.04". Без команди від юзера | Залежить від 4.1 |
| 4.8 | **Smart Notification Queue** | Черга повідомлень з пріоритетом замість перезапису табло. OWL хоче сказати 3 речі → показує по черзі. Критичні зверху | Середня |

## 📅 Календар (затверджено 30.03.2026)

**Розміщення:** кнопка 📅 між лічильниками Задачі і Звички у вкладці Продуктивність. Календар з вкладки Я — прибрати (лишити тільки кільця активності).

**UI флоу:**
1. Тап 📅 → модальне вікно з місячним календарем
2. Дні з задачами/подіями мають кольоровий маркер (🔴 критичний, 🟠 важливий, звичайний)
3. Тап на день → zoom-анімація: клітинка розширюється, дата заповнює екран → fade → список записів дня
4. У списку дня: задачі + події, кнопка `+ Додати`, редагування існуючих

**Типи записів:**
- Задача з дедлайном (`dueDate` на існуючій задачі)
- Подія — без чекбоксу, просто дата/час + нотатка (`nm_events`, новий ключ)

**Event структура:**
```javascript
{ id, title, date, time?, priority: 'normal'|'important'|'critical', notes? }
```

**Розширення Task:**
```javascript
{ ...існуюче..., dueDate?: timestamp, dueTime?: 'HH:MM', priority?: 'normal'|'important'|'critical' }
```

**Нагадування — система пріоритетів:**

| Пріоритет | Коли | Механізм |
|-----------|------|----------|
| 🔴 Критичний | За 3 дні + за день + в день | Червона картка закріплена ЗВЕРХУ Inbox |
| 🟠 Важливий | За день + в день | Помаранчева картка зверху Inbox |
| 🟡 Звичайний | В день | OWL board згадує проактивно |

Картку можна закрити свайпом. Поп-апів немає — закріплена картка в Inbox достатньо помітна і не лякає.

**getAIContext() — новий блок:**
```
Заплановано на найближчі 7 днів:
- [ID:xxx] Постірати одяг — 31 бер.
- [EVENT] Платіж за паркінг — 1 квіт. 🔴 критичний
```

**Порядок реалізації:**
1. `dueDate` + `priority` в Task. Кнопка 📅 між лічильниками.
2. Calendar modal + zoom-анімація. Маркери на днях.
3. `nm_events` — нові події. Форма додавання.
4. Закріплені картки-нагадування в Inbox.
5. `getAIContext()` + OWL нагадування по `dueDate`.
