## 🗂️ OWL board — живий міні-чат (уточнено 01.04.2026)

**Зараз:** пасивний слайдер (3 повідомлення, чіпи = теги без дії).

**Концепція:** OWL board стає активним міні-чатом. OWL ініціює — користувач відповідає.

### Два окремих чати
| | OWL міні-чат (зверху) | Основний чат (знизу) |
|--|--|--|
| Хто ініціює | OWL | Користувач |
| Призначення | Діалог з агентом | Записи в inbox |
| Сховище | `nm_owl_chat` | `nm_chat_inbox` |

### 5 станів OWL board
1. **Тихий** — сіра крапка, остання фраза курсивом, "нема нових"
2. **Нове повідомлення** — жовта крапка, текст превью, "тап щоб відповісти ↓"
3. **Розгорнутий** — міні-чат відкритий: бульбашки + чіпи + мінi-input
4. **OWL друкує** — "•••" анімація, поле вводу заблоковане
5. **Підтвердження** — зелений банер на 2 сек після дії

### Чіпи = текстова відповідь
```
Тап 'Так, о 7:00'
→ відправляється як повідомлення в nm_owl_chat
→ OWL '•••' → обробляє → відповідає і виконує дію
→ 'Поставив. Стрік ще живий 💪'
```

### Ініціативність агента
- Самостійно починає розмову на основі тригерів (існуюча логіка `checkOwlBoardTrigger`)
- Говорить як жива людина зі своєю роллю (coach/partner/mentor)
- Ніколи: "Звичайно!", "Чудово!", не починає з "Я"
- Відповідь 1-2 речення, конкретно і по-людськи

### Що залишається без змін
- Тригери (`checkOwlBoardTrigger`) і контекст (`getOwlBoardContext`) — не чіпати
- Ритм генерації (кожні 3 хв, активний 7:00-23:00)
- Антиповтор `nm_owl_board_said`
- Основний чат знизу — незалежний

### Що змінюється в коді
- `index.html`: OWL board HTML → компактний превью + розгорнутий міні-чат
- `app-ai-chat.js`: `renderOwlBoard()` → `renderOwlBoardCompact()` + `renderOwlChatMessages()`; нові функції: `toggleOwlChat()`, `sendOwlReply(text)`, `showOwlTyping()`
- `app-ai-core.js`: нова `callOwlChat(history)` — короткі відповіді, той самий action-формат
- `style.css`: стилі для міні-чату, чіпи-кнопки, "•••" анімація

### Ще не вирішено
- Дизайн конкретних елементів (аватар, розміри, кольори) — окрема сесія

> ⚠️ Середній обсяг. Окрема сесія імплементації.

## 🗂️ Inbox стрічка — редизайн (затверджено 30.03.2026)

**Зараз:** великі картки з іконкою + текст + тег → не інформативно, займає багато місця.

**Нове — компактна стрічка:**
- Кольорова крапка зліва (🔵 задача, 🟡 фінанси, 🟢 нотатка, 🟣 момент)
- Текст (1 рядок, truncate) + тег справа
- Датові сепаратори між днями: "СЬОГОДНІ", "ВЧОРА", "25 БЕР."
- Тап на запис → відкриває деталі (TBD)

**Закріплені картки нагадувань** (зверху стрічки, під OWL board):
- 🔴 Критичні (червоний фон) — з'являються за 3 дні до події
- 🟠 Важливі (помаранчевий фон) — за день до події
- Закрити свайпом або ×

### Уточнені рішення (01.04.2026)

**Свайп-видалення:** залишається як є.

**Тап на запис:** текст в стрічці обрізається до 1 рядка з "..." в кінці. Тап → повний текст розгортається (в тій самій картці або превью-блоці — вирішити при реалізації).

**Кольори категорій:** всі категорії мають свій колір. Вже визначено в `CAT_DOT_BG` (`app-inbox.js`):
- 🔵 Задача · 🟢 Нотатка · 🟡 Ідея · 🟠 Фінанси · 🔵 Подія · 🟢 Звичка

**Крапка = пілюль тегу:** крапка зліва і тег справа — один і той самий колір категорії. Окрема схема не потрібна.

**Розгортання по тапу:** картка розширюється на місці (текст збільшується в тій самій картці, повторний тап — згортається).

**Закріплені картки нагадувань:** повна синхронізація з усіма вкладками — Календар, Задачі, Звички, Здоров'я, Проекти, Фінанси. Будь-який дедлайн або подія з будь-якої вкладки може створити закріплену картку в Inbox.

> ⚠️ Окрема сесія імплементації.

## 📅 Календар (затверджено 30.03.2026)

**Розміщення:** кнопка 📅 між лічильниками Задачі і Звички у вкладці Продуктивність. Календар з вкладки Я — прибрати (лишити тільки кільця активності).

**UI флоу:**
1. Тап 📅 → модальне вікно з місячним календарем
2. Дні з задачами/подіями мають кольоровий маркер (🔴 критичний, 🟠 важливий, звичайний)
3. Тап на день → zoom-анімація: клітинка розширюється, дата заповнює екран → fade → список записів дня
4. У списку дня: задачі + події, кнопка `+ Додати`, редагування існуючих

**Типи записів:**
- Задача з дедлайном (`dueDate` на існуючій задачі)
- Подія — без чекбоксу, просто дата/час + нотатка (`nm_events`, новий ключ)

**Event структура:**
```javascript
{ id, title, date, time?, priority: 'normal'|'important'|'critical', notes? }
```

**Розширення Task:**
```javascript
{ ...існуюче..., dueDate?: timestamp, dueTime?: 'HH:MM', priority?: 'normal'|'important'|'critical' }
```

**Нагадування — система пріоритетів:**

| Пріоритет | Коли | Механізм |
|-----------|------|----------|
| 🔴 Критичний | За 3 дні + за день + в день | Червона картка закріплена ЗВЕРХУ Inbox |
| 🟠 Важливий | За день + в день | Помаранчева картка зверху Inbox |
| 🟡 Звичайний | В день | OWL board згадує проактивно |

Картку можна закрити свайпом. Поп-апів немає — закріплена картка в Inbox достатньо помітна і не лякає.

**getAIContext() — новий блок:**
```
Заплановано на найближчі 7 днів:
- [ID:xxx] Постірати одяг — 31 бер.
- [EVENT] Платіж за паркінг — 1 квіт. 🔴 критичний
```

**Порядок реалізації:**
1. `dueDate` + `priority` в Task. Кнопка 📅 між лічильниками.
2. Calendar modal + zoom-анімація. Маркери на днях.
3. `nm_events` — нові події. Форма додавання.
4. Закріплені картки-нагадування в Inbox.
5. `getAIContext()` + OWL нагадування по `dueDate`.

## 🦉 Живий OWL (обговорення 30.03.2026)

**Проблема зараз:** OWL board виглядає як "стрічка новин біржі". Агент пасивний — тільки відповідає, не ініціює.

**Нова концепція — три ролі OWL:**

1. **Допитливий** — сам коментує записи користувача
   - "Бачу ти додав задачу 3 дні тому. Вже зробив?"
   - "Що таке 'автошоу'? Це подія? Хочеш нагадування?"

2. **Вчитель** — показує як користуватись додатком
   - "Спробуй написати 'купити молоко завтра' — я створю задачу з датою"
   - "Знаєш що свайп вліво видаляє запис?"

3. **Ініціатор** — починає розмову першим
   - Ранок: "Доброго ранку! Що сьогодні найважливіше?"
   - Після стріку: "5 днів поспіль! Що допомагає тримати стрік?"

**Візуально:** аватар OWL + бульбашка-повідомлення, анімація "друкує...", розмовний тон.

**Inbox:** окреме обговорення (картки vs чат-фокус).

> ⚠️ Великий редизайн. Окрема сесія планування перед імплементацією.
